# iOS

Languine støtter iOS Localizable.strings-filer, det standard lokaliseringsformatet for iOS-applikasjoner. Dette formatet håndterer grunnleggende nøkkel-verdi tekstpar og støtter kommentarer for å gi kontekst til oversettere.

---

## Sette opp

Først, sørg for at du har en languine.config.json eller languine.config.ts konfigurasjonsfil i rotmappen til prosjektet ditt. Her er et eksempel:

```json
{
  "locale": {
    "source": "en",
    "targets": ["sv", "de", "fr"]
  },
  "files": {
    "ios": {
      "include": ["[locale].lproj/Localizable.strings"]
    }
  }
}
```

## Oversette

Med konfigurasjonen din satt opp, kjør:

```bash
npx languine@latest translate
```

Når du kjører denne kommandoen, vil Languine:

- Laste inn din kilde Localizable.strings-fil (f.eks. en.lproj/Localizable.strings)
- Oppdage eventuelle nye eller endrede oversettelsesstrenger
- Generere oversettelser for målspråkene dine
- Opprette eller oppdatere målspråkfilene (f.eks. sv.lproj/Localizable.strings)
- Beholde eventuelle eksisterende kommentarer og formatering
