# GitHub Actions

ជាមួយនឹងសកម្មភាព GitHub អ្នកអាចបម្លែងកូដរបស់អ្នកបានយ៉ាងងាយស្រួលដោយគ្រាន់តែបញ្ចូលកូដរបស់អ្នកទៅក្នុងឃ្លាំង សកម្មភាព GitHub នឹងបម្លែងកូនសោថ្មីរបស់អ្នក បង្កើតសំណើសម្រង់ឬបញ្ជូនការផ្លាស់ប្តូររបស់អ្នក។
នេះអាចសន្សំសំចៃពេលវេលានិងកម្លាំងរបស់អ្នកបានយ៉ាងច្រើន ខណៈពេលដែលរក្សាទុកគុណភាពខ្ពស់នៃការបម្លែង។

---

## ការតំឡើង

ដើម្បីប្រើសកម្មភាព Languine GitHub ក្នុងឃ្លាំងរបស់អ្នក អ្នកត្រូវតែ:

1. កំណត់សម្ងាត់ឃ្លាំងរបស់អ្នកជាមួយអត្តសញ្ញាណ Languine របស់អ្នក
2. បង្កើតឯកសារប្រតិបត្តិការនៅក្នុងឃ្លាំងរបស់អ្នក

### ការកំណត់សម្ងាត់ឃ្លាំង

បន្ថែមសម្ងាត់ដូចខាងក្រោមនៅក្នុងការកំណត់ឃ្លាំងរបស់អ្នក (ការកំណត់ > សម្ងាត់ និងអថេរ > សកម្មភាព):

- `LANGUINE_API_KEY`: សម្ងាត់ API Languine របស់អ្នក
- `LANGUINE_PROJECT_ID`: អត្តសញ្ញាណគម្រោង Languine របស់អ្នក

អ្នកអាចស្វែងរកតម្លៃទាំងនេះនៅក្នុងផ្ទាំងគ្រប់គ្រង Languine របស់អ្នកក្រោមការកំណត់គម្រោង។



### អថេរ​បរិស្ថាន

សកម្មភាព GitHub ប្រើអថេរបរិស្ថានដូចខាងក្រោម:
- `LANGUINE_API_KEY` (ត្រូវការ): សម្ងាត់ API Languine របស់អ្នកសម្រាប់ការផ្ទៀងផ្ទាត់
- `LANGUINE_PROJECT_ID` (ត្រូវការ): អត្តសញ្ញាណគម្រោង Languine របស់អ្នក
- `GITHUB_TOKEN` (ត្រូវការ): ផ្ដល់ដោយ GitHub យ៉ាងស្វ័យប្រវត្តិ ប្រើសម្រាប់បង្កើត PRs

### ឧទាហរណ៍ប្រតិបត្តិការនៅក្នុងឯកសារ

បង្កើតឯកសារថ្មី .github/workflows/languine.yml នៅក្នុងឃ្លាំងរបស់អ្នកជាមួយមាតិកាដូចខាងក្រោម:

```ts title=".github/workflows/release.yml"
name: បង្ហោះគេហទំព័រ

on:
  push:
    branches:
      - '**'
    paths:
      - apps/web/**

jobs:
  translate:
    runs-on: ubuntu-latest
    permissions:
      contents: write
      pull-requests: write
    steps:
      - uses: actions/checkout@v4
      
      - name: សកម្មភាព Languine
        uses: languine/languine-action@v1
        env:
           GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
        with:
          api-key: ${{ secrets.LANGUINE_API_KEY }}
          project-id: ${{ secrets.LANGUINE_PROJECT_ID }}
```
 
 សកម្មភាពនេះនឹង:
- ប្រតិបត្តិការនៅពេលមានការបញ្ចូលទៅកាន់សាខាមេ
- ប្រតិបត្តិការនៅពេលមានសំណើសម្រង់ទៅកាន់សាខាមេ
- អាចត្រូវបានប្រើប្រាស់ដោយដៃពីផ្ទាំងសកម្មភាព
- ចុះឯកសារកូដរបស់អ្នក និងដំណើរការបម្លែង Languine
- បង្កើតសំណើសម្រង់ជាមួយការបម្លែងថ្មីណាមួយ (អាចកំណត់)

#### ជម្រើសថតការងារ

ជម្រើស `working-directory` អនុញ្ញាតឱ្យអ្នកកំណត់ទីតាំងដែលឯកសារបម្លែងរបស់អ្នកស្ថិតនៅ។ នេះមានប្រយោជន៍យ៉ាងខ្លាំងនៅក្នុង monorepos ដែលការបម្លែងរបស់អ្នកអាចស្ថិតនៅក្នុងកញ្ចប់ឬថតកម្មវិធីជាក់លាក់។ ឧទាហរណ៍:

#### ជម្រើសសំណើសម្រង់

ជម្រើស `create-pull-request` គ្រប់គ្រងថាសកម្មភាពបង្កើតសំណើសម្រង់ជាមួយការផ្លាស់ប្តូរឬបញ្ជូនពួកវាទៅកាន់សាខា៖

```typescript title=".github/workflows/languine.yml"
with:
  create-pull-request: true # បង្កើតសំណើសម្រង់ជាមួយការផ្លាស់ប្តូរ
```

នៅពេលកំណត់ទៅ `true` សកម្មភាពនឹង:
- បង្កើតសាខាថ្មីជាមួយការផ្លាស់ប្តូរ
- បង្កើតសំណើសម្រង់សម្រាប់ការត្រួតពិនិត្យ
- បន្ថែមស្លាកនិងការពិពណ៌នាដែលសមស្រប

នៅពេលកំណត់ទៅ `false` (លំនាំដើម) សកម្មភាពនឹង:
- បញ្ជូនការផ្លាស់ប្តូរទៅកាន់សាខាបច្ចុប្បន្ន
- បញ្ជូនការផ្លាស់ប្តូរដោយមិនបង្កើតសំណើសម្រង់
