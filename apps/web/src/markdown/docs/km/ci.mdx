# CI/CD

Languine ផ្តល់នូវការបញ្ចូល CI/CD ដើម្បីជួយអ្នកបកប្រែកម្មវិធីរបស់អ្នក។

---

## វិធីដែលវាធ្វើការ

ដំណើរការប្រព័ន្ធ CI/CD នៅក្នុង Languine តាមដានដំណើរការដែលមានរចនាសម្ព័ន្ធដើម្បីស្វ័យប្រវត្តិកម្មវិធីបកប្រែឱ្យមានការអាប់ដេតនៅក្នុងកម្មវិធីរបស់អ្នក៖

### 1. ការចាប់ផ្តើមសកម្មភាព
- ដំណើរការនេះចាប់ផ្តើមនៅពេលដែលសកម្មភាពត្រូវបានបញ្ចូល
- កំណត់ការកំណត់ Git និងអត្តសញ្ញាណដែលចាំបាច់

### 2. ការប្រតិបត្តិ Languine Engine
- ម៉ាស៊ីន Languine ប្រតិបត្តិមាតិកានៃកម្មវិធីរបស់អ្នក
- គ្រប់គ្រងការបកប្រែដោយប្រើការកំណត់ដែលបានកំណត់

### 3. ការត្រួតពិនិត្យការផ្លាស់ប្តូរ
ប្រព័ន្ធត្រួតពិនិត្យសម្រាប់ការផ្លាស់ប្តូរបកប្រែណាមួយ ហើយតាមដានមួយក្នុងពីរផ្លូវ៖

**នៅពេលដែលមានការផ្លាស់ប្តូរ៖**
- ការផ្លាស់ប្តូរត្រូវបានបញ្ជូនទៅកាន់ឃ្លាំង
- ការផ្លាស់ប្តូរត្រូវបានបញ្ចូលទៅកាន់សាខាដែលបានកំណត់
- ការស្នើសុំ Pull Request (PR) ត្រូវបានបង្កើត

**នៅពេលដែលមិនមានការផ្លាស់ប្តូរ៖**
- ដំណើរការបញ្ចប់ដោយមិនបង្កើត PR

### 4. ការបង្កើត Pull Request
នៅពេលដែលមានការផ្លាស់ប្តូរត្រូវបានរកឃើញ, Pull Request ត្រូវបានបង្កើតជាមួយ៖

**មាតិកា PR:**
- ចំណងជើងច្បាស់ដែលបង្ហាញពីការអាប់ដេតបកប្រែ
- ការពិពណ៌នាយ៉ាងលម្អិតអំពីការផ្លាស់ប្តូរ
- ការបញ្ជាក់អំពីការត្រួតពិនិត្យគុណភាព
- សេចក្តីណែនាំសម្រាប់ការត្រួតពិនិត្យ
- តំណទៅឯកសារនិងការគាំទ្រ

**ការពិពណ៌នាអំពី PR រួមមាន៖**
- សង្ខេបនៃការអាប់ដេតបកប្រែ
- បញ្ជីនៃអ្វីដែលបានផ្លាស់ប្តូរ (ការបកប្រែដែលបានអាប់ដេត, សមាសភាពពាក្យ)
- ជំហានបន្ទាប់សម្រាប់អ្នកត្រួតពិនិត្យ
- តំណទៅឯកសារនិងធនធានការគាំទ្រ
