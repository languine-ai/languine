# CLI

Languine 提供了一個 CLI 來幫助您翻譯應用程式。

## 命令概覽

### translate

`translate` 命令用於翻譯您的應用程式。

```bash title="Terminal"
# 開始翻譯您的應用程式
languine translate

# 選項：
#   --source-locale    指定來源語言（預設：en）
#   --target-locale    指定要翻譯的目標語言
#   --config          您的 languine.config.js 檔案的路徑
```

### init

```bash title="Terminal"
# 初始化一個新的 Languine 專案
languine init

# 這將：
# - 創建一個 languine.config.js 檔案
# - 設置您的專案結構
# - 初始化必要的目錄
```

### status

```bash title="Terminal"
# 檢查翻譯的狀態
languine status

# 顯示：
# - 當前翻譯進度
# - 缺失的翻譯
# - 翻譯統計
```

### validate

```bash title="Terminal"
# 驗證您的翻譯檔案
languine validate

# 檢查：
# - 檔案結構
# - 缺失的鍵
# - 格式一致性
```

### export

```bash title="Terminal"
# 匯出您的翻譯
languine export

# 選項：
#   --format          匯出格式（json, yaml 等）
#   --output         輸出目錄
```

### import

```bash title="Terminal"
# 從外部檔案匯入翻譯
languine import

# 選項：
#   --source         來源檔案或目錄
#   --format         匯入格式（json, yaml 等）
```

## 全域選項

所有命令都支持以下全域選項：

```bash title="Terminal"
# 所有命令可用的全域選項
languine [command] [options]

# 常見選項：
#   --help, -h       顯示幫助資訊
#   --version, -v    顯示版本資訊
#   --verbose        啟用詳細日誌
#   --config         指定配置檔案路徑
```
