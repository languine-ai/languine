# 配置

Languine 支援 TypeScript 和 JSON 配置檔案來管理您的翻譯設置。您可以選擇使用 languine.config.ts 來進行 TypeScript 配置，或使用 languine.json 來進行 JSON 格式配置。這兩個檔案都定義了您的源語言和目標語言，以及需要翻譯的檔案。

---

## 基本配置

以下是兩種格式的基本範例：

```typescript title="languine.config.ts"
import { defineConfig } from "languine";

export default defineConfig({
  locale: {
    source: "en",
    targets: ["es", "fr", "de"],
  },
  files: {
    json: {
      include: ["locales/[locale].json"],
    },
  },
});
```

```json title="languine.json"
{
  "locale": {
    "source": "en",
    "targets": ["es", "fr", "de"]
  },
  "files": {
    "json": {
      "include": ["locales/[locale].json"]
    }
  }
}
```

這兩種格式支援相同的配置選項。選擇 TypeScript 以獲得更好的類型檢查和 IDE 支援，或選擇 JSON 以獲得簡單性和更廣泛的工具相容性。

讓我們來分析這些檔案的不同部分。

---

## 語言配置

語言部分定義了您的翻譯語言：

- source: 您的主要內容創建語言（例如，"en" 代表英語）
- targets: 您希望翻譯成的語言陣列（例如， ["es", "fr", "de"] 代表西班牙語、法語和德語）

Languine 支援多種語言代碼，包括：
- 簡單代碼：en, es, fr, de
- 區域特定代碼：pt-BR, zh-CN, zh-TW

## 檔案配置

檔案部分定義了 Languine 應該翻譯哪些檔案。您可以配置多種檔案類型和模式：

---

### JSON 檔案
```typescript
files: {
  json: {
    include: ["locales/[locale].json"],
  },
}
```

---

### Markdown/MDX 檔案
```typescript
files: {
  md: {
    include: ["docs/[locale]/*.md"],
  },
  mdx: {
    include: ["blog/[locale]/*.mdx"],
  },
}
```

---

### 行動平台檔案

對於 iOS (Xcode)：
```typescript
files: {
  "xcode-xcstrings": {
    include: ["Example/Localizable.xcstrings"],
  },
  // 或者對於傳統的 .strings 檔案
  "xcode-strings": {
    include: ["Example/[locale].lproj/Localizable.strings"],
  },
}
```

---

對於 Android：
```typescript
files: {
  android: {
    include: ["locales/[locale].xml"],
  },
}
```

## 檔案模式

檔案模式中的 [locale] 佔位符在翻譯過程中會自動替換為適當的語言代碼。例如：

- locales/[locale].json 變成：
  - locales/en.json
  - locales/es.json
  - locales/fr.json

您可以為每種檔案類型包含多個模式並使用 glob 模式：
```typescript
files: {
  json: {
    include: [
      "src/locales/[locale].json",
      "src/components/[locale]/*.json"
    ],
  },
}
```

## 專案 ID (可選)

如果您使用 Languine 的雲端服務，您可以在配置中包含您的專案 ID：

```typescript
export default defineConfig({
  projectId: "prj_your_project_id",
  locale: {
    // ... 語言配置
  },
  files: {
    // ... 檔案配置
  },
});
```

## 翻譯狀態追蹤

Languine 自動創建並維護 languine.lock 檔案以追蹤您的翻譯狀態。此檔案：

- 追蹤已翻譯的內容
- 幫助識別當源內容變更時需要更新的內容
- 應該提交到您的版本控制系統

您不需要手動編輯鎖定檔案 - 當您運行翻譯命令時，Languine 會自動管理它。

## 最佳實踐

1. **版本控制**：始終將 languine.config.ts 和 languine.lock 都提交到您的儲存庫
2. **檔案組織**：保持您的翻譯檔案在一致的目錄結構中
3. **語言代碼**：使用標準語言代碼以確保相容性
4. **模式**：使用特定的包含模式以避免翻譯不必要的檔案
5. **專案 ID**：保持您的專案 ID 安全，並在使用雲端服務時不要公開分享
