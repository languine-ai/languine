# CI/CD

Languine 提供 CI/CD 整合，幫助您翻譯應用程式。

---

## 運作方式

Languine 的 CI/CD 工作流程遵循結構化過程，自動化應用程式中的翻譯更新：

### 1. 行動初始化
- 當觸發行動時，工作流程開始
- 設置必要的 Git 配置和憑證

### 2. Languine 引擎處理
- Languine 引擎處理您應用程式的內容
- 使用配置的設置處理翻譯任務

### 3. 變更檢測
系統檢查是否有任何翻譯變更，並遵循兩條路徑之一：

**當發現變更時：**
- 變更被提交到儲存庫
- 變更被推送到指定的分支
- 創建拉取請求 (PR)

**當未發現變更時：**
- 工作流程完成，未創建 PR

### 4. 拉取請求創建
當檢測到變更時，創建一個拉取請求，內容包括：

**PR 內容：**
- 清晰的標題，指示翻譯更新
- 詳細的變更描述
- 質量檢查確認
- 審查指示
- 文檔和支持的鏈接

**PR 描述包括：**
- 翻譯更新的摘要
- 變更列表（更新的翻譯、術語一致性）
- 審查者的後續步驟
- 文檔和支持資源的鏈接
