# CI/CD

Languine, uygulamanızı çevirmek için CI/CD entegrasyonu sağlar.

---

## Nasıl Çalışır

Languine'deki CI/CD iş akışı, uygulamanızdaki çeviri güncellemelerini otomatikleştirmek için yapılandırılmış bir süreç izler:

### 1. Eylem Başlatma
- İş akışı, eylem tetiklendiğinde başlar
- Gerekli Git yapılandırması ve kimlik bilgileri ayarlanır

### 2. Languine Motoru İşleme
- Languine motoru, uygulamanızın içeriğini işler
- Yapılandırılmış ayarları kullanarak çeviri görevlerini yönetir

### 3. Değişiklik Tespiti
Sistem, herhangi bir çeviri değişikliği olup olmadığını kontrol eder ve iki yoldan birini takip eder:

**Değişiklik Bulunduğunda:**
- Değişiklikler depoya kaydedilir
- Değişiklikler belirtilen dalga gönderilir
- Bir Pull Request (PR) oluşturulur

**Değişiklik Bulunmadığında:**
- İş akışı, PR oluşturmadan tamamlanır

### 4. Pull Request Oluşturma
Değişiklikler tespit edildiğinde, aşağıdakilerle bir Pull Request oluşturulur:

**PR İçeriği:**
- Çeviri güncellemelerini belirten net bir başlık
- Değişikliklerin ayrıntılı açıklaması
- Kalite kontrol onayları
- İnceleme için talimatlar
- Belgeler ve destek için bağlantılar

**PR Açıklaması Şunları İçerir:**
- Çeviri güncellemelerinin özeti
- Nelerin değiştiği listesi (güncellenmiş çeviriler, terminoloji tutarlılığı)
- İnceleyiciler için sonraki adımlar
- Belgeler ve destek kaynaklarına bağlantılar
