# GitHub Actions

GitHub eylemi ile kodunuzu sadece depoya iterek kolayca çevirebilirsiniz, GitHub eylemi yeni anahtarlarınızı çevirecek, ya bir pull request oluşturacak ya da değişikliklerinizi taahhüt edecektir.
Bu, yüksek kaliteli çevirileri korurken size çok zaman ve çaba kazandırır.

---

## Kurulum

Depomuzda Languine GitHub Eylemini kullanmak için şunları yapmalısınız:

1. Languine kimlik bilgilerinizle depo gizli anahtarlarınızı ayarlayın
2. Depoda bir iş akışı dosyası oluşturun

### Depo Gizli Anahtarlarını Ayarlama

Depo ayarlarınızda (Ayarlar > Gizli Anahtarlar ve Değişkenler > Eylemler) aşağıdaki gizli anahtarları ekleyin:

- `LANGUINE_API_KEY`: Languine API anahtarınız
- `LANGUINE_PROJECT_ID`: Languine proje kimliğiniz

Bu değerleri Languine kontrol panelinizde Proje Ayarları altında bulabilirsiniz.



### Ortam Değişkenleri

GitHub Eylemi aşağıdaki ortam değişkenlerini kullanır:
- `LANGUINE_API_KEY` (Gerekli): Kimlik doğrulama için Languine API anahtarınız
- `LANGUINE_PROJECT_ID` (Gerekli): Languine proje kimliğiniz 
- `GITHUB_TOKEN` (Gerekli): GitHub tarafından otomatik olarak sağlanır, PR'ler oluşturmak için kullanılır

### Örnek İş Akışı

Depoda aşağıdaki içeriğe sahip .github/workflows/languine.yml adında yeni bir dosya oluşturun:

```ts title=".github/workflows/release.yml"
name: Web Sitesini Yayınla

on:
  push:
    branches:
      - '**'
    paths:
      - apps/web/**

jobs:
  translate:
    runs-on: ubuntu-latest
    permissions:
      contents: write
      pull-requests: write
    steps:
      - uses: actions/checkout@v4
      
      - name: Languine Eylemi
        uses: languine/languine-action@v1
        env:
           GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
        with:
          api-key: ${{ secrets.LANGUINE_API_KEY }}
          project-id: ${{ secrets.LANGUINE_PROJECT_ID }}
```
 
 Bu iş akışı şunları yapacaktır:
- Ana dalda yapılan itmelere yanıt verecektir
- Ana dalda yapılan pull request'lere yanıt verecektir
- Eylemler sekmesinden manuel olarak tetiklenebilir
- Kodunuzu kontrol edecek ve Languine çeviri sürecini çalıştıracaktır
- Herhangi bir yeni çeviri ile bir pull request oluşturacaktır (ayarlarla değiştirilebilir)

#### Pull Request Seçeneği

`create-pull-request` seçeneği, eylemin değişikliklerle bir pull request oluşturup oluşturmayacağını veya bunları doğrudan dala taahhüt edip etmeyeceğini kontrol eder:

```typescript title=".github/workflows/languine.yml"
with:
  create-pull-request: true # değişikliklerle bir pull request oluştur
```

`true` olarak ayarlandığında, eylem şunları yapacaktır:
- Değişikliklerle yeni bir dal oluşturacaktır
- İnceleme için bir pull request oluşturacaktır
- Uygun etiketler ve açıklama ekleyecektir

`false` (varsayılan) olarak ayarlandığında, eylem şunları yapacaktır:
- Değişiklikleri doğrudan mevcut dala taahhüt edecektir
- Pull request oluşturmadan değişiklikleri itecektir
