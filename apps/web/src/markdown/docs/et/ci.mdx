# CI/CD

Languine pakub CI/CD integreerimist, et aidata teil tõlkida oma rakendust.

---

## Kuidas see töötab

Languine'i CI/CD töövoog järgib struktureeritud protsessi, et automatiseerida tõlkeuuendusi teie rakenduses:

### 1. Tegevuse algatamine
- Töövoog algab, kui tegevus käivitatakse
- Seab üles vajalikud Git konfiguratsioonid ja volitused

### 2. Languine'i mootori töötlemine
- Languine'i mootor töötleb teie rakenduse sisu
- Teeb tõlketöid konfigureeritud seadete abil

### 3. Muudatuste tuvastamine
Süsteem kontrollib tõlke muudatusi ja järgib ühte kahest teest:

**Kui muudatused on leitud:**
- Muudatused salvestatakse hoidlasse
- Muudatused edastatakse määratud harule
- Koostatakse Pull Request (PR)

**Kui muudatusi ei leita:**
- Töövoog lõpeb ilma PR-i loomata

### 4. Pull Requesti loomine
Kui muudatused on tuvastatud, luuakse Pull Request koos:

**PR sisu:**
- Selge pealkiri, mis näitab tõlkeuuendusi
- Üksikasjalik muudatuste kirjeldus
- Kvaliteedikontrolli kinnitused
- Ülevaatamise juhised
- Lingid dokumentatsiooni ja toe juurde

**PR kirjeldus sisaldab:**
- Tõlkeuuenduste kokkuvõte
- Loetelu muudatustest (uuendatud tõlked, terminoloogia järjepidevus)
- Järgmised sammud ülevaatajate jaoks
- Lingid dokumentatsiooni ja toe ressursside juurde
