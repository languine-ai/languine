# CLI

Languine มี CLI เพื่อช่วยให้คุณแปลแอปพลิเคชันของคุณ

## ภาพรวมคำสั่ง

### translate

คำสั่ง `translate` ใช้สำหรับแปลแอปพลิเคชันของคุณ

```bash title="Terminal"
# เริ่มการแปลแอปพลิเคชันของคุณ
languine translate

# ตัวเลือก:
#   --source-locale    ระบุแหล่งที่มาของภาษา (ค่าเริ่มต้น: en)
#   --target-locale    ระบุภาษาที่ต้องการแปลไป
#   --config          เส้นทางไปยังไฟล์ languine.config.js ของคุณ
```

### init

```bash title="Terminal"
# เริ่มโครงการ Languine ใหม่
languine init

# สิ่งนี้จะ:
# - สร้างไฟล์ languine.config.js
# - ตั้งค่าโครงสร้างโครงการของคุณ
# - เริ่มต้นไดเรกทอรีที่จำเป็น
```

### status

```bash title="Terminal"
# ตรวจสอบสถานะของการแปลของคุณ
languine status

# แสดง:
# - ความก้าวหน้าของการแปลในปัจจุบัน
# - การแปลที่ขาดหายไป
# - สถิติการแปล
```

### validate

```bash title="Terminal"
# ตรวจสอบไฟล์การแปลของคุณ
languine validate

# ตรวจสอบ:
# - โครงสร้างไฟล์
# - คีย์ที่ขาดหายไป
# - ความสอดคล้องของรูปแบบ
```

### export

```bash title="Terminal"
# ส่งออกการแปลของคุณ
languine export

# ตัวเลือก:
#   --format          รูปแบบการส่งออก (json, yaml, ฯลฯ)
#   --output         ไดเรกทอรีผลลัพธ์
```

### import

```bash title="Terminal"
# นำเข้าการแปลจากไฟล์ภายนอก
languine import

# ตัวเลือก:
#   --source         ไฟล์หรือไดเรกทอรีต้นทาง
#   --format         รูปแบบการนำเข้า (json, yaml, ฯลฯ)
```

## ตัวเลือกทั่วโลก

คำสั่งทั้งหมดสนับสนุนตัวเลือกทั่วโลกต่อไปนี้:

```bash title="Terminal"
# ตัวเลือกทั่วโลกที่มีให้สำหรับคำสั่งทั้งหมด
languine [command] [options]

# ตัวเลือกทั่วไป:
#   --help, -h       แสดงข้อมูลช่วยเหลือ
#   --version, -v    แสดงข้อมูลเวอร์ชัน
#   --verbose        เปิดใช้งานการบันทึกแบบละเอียด
#   --config         ระบุเส้นทางไฟล์ config
```
