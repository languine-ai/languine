# CI/CD

Languine มีการรวม CI/CD เพื่อช่วยให้คุณแปลแอปพลิเคชันของคุณ

---

## วิธีการทำงาน

กระบวนการ CI/CD ใน Languine จะปฏิบัติตามขั้นตอนที่มีโครงสร้างเพื่อทำให้การอัปเดตการแปลในแอปพลิเคชันของคุณเป็นอัตโนมัติ:

### 1. การเริ่มต้นการกระทำ
- กระบวนการเริ่มต้นเมื่อมีการกระตุ้นการกระทำ
- ตั้งค่าคอนฟิกูเรชัน Git และข้อมูลประจำตัวที่จำเป็น

### 2. การประมวลผลของ Languine Engine
- Languine engine จะประมวลผลเนื้อหาของแอปพลิเคชันของคุณ
- จัดการงานการแปลโดยใช้การตั้งค่าที่กำหนด

### 3. การตรวจจับการเปลี่ยนแปลง
ระบบจะตรวจสอบการเปลี่ยนแปลงการแปลและติดตามหนึ่งในสองเส้นทาง:

**เมื่อพบการเปลี่ยนแปลง:**
- การเปลี่ยนแปลงจะถูกบันทึกในที่เก็บ
- การเปลี่ยนแปลงจะถูกส่งไปยังสาขาที่กำหนด
- Pull Request (PR) จะถูกสร้างขึ้น

**เมื่อไม่พบการเปลี่ยนแปลง:**
- กระบวนการจะเสร็จสิ้นโดยไม่สร้าง PR

### 4. การสร้าง Pull Request
เมื่อมีการตรวจพบการเปลี่ยนแปลง จะมีการสร้าง Pull Request พร้อม:

**เนื้อหา PR:**
- ชื่อที่ชัดเจนระบุการอัปเดตการแปล
- คำอธิบายรายละเอียดของการเปลี่ยนแปลง
- การยืนยันการตรวจสอบคุณภาพ
- คำแนะนำสำหรับการตรวจสอบ
- ลิงก์ไปยังเอกสารและการสนับสนุน

**คำอธิบาย PR รวมถึง:**
- สรุปการอัปเดตการแปล
- รายการสิ่งที่เปลี่ยนแปลง (การแปลที่อัปเดต ความสอดคล้องของคำศัพท์)
- ขั้นตอนถัดไปสำหรับผู้ตรวจสอบ
- ลิงก์ไปยังเอกสารและแหล่งข้อมูลการสนับสนุน
