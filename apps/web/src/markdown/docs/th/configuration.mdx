# การตั้งค่า

Languine รองรับไฟล์การตั้งค่าทั้ง TypeScript และ JSON เพื่อจัดการการตั้งค่าการแปลของคุณ คุณสามารถเลือกใช้ languine.config.ts สำหรับ TypeScript หรือ languine.json สำหรับรูปแบบ JSON ไฟล์ทั้งสองนี้กำหนดภาษาต้นทางและภาษาที่ต้องการแปล รวมถึงไฟล์ที่ต้องแปล

---

## การตั้งค่าพื้นฐาน

นี่คือตัวอย่างพื้นฐานในทั้งสองรูปแบบ:

```typescript title="languine.config.ts"
import { defineConfig } from "languine";

export default defineConfig({
  locale: {
    source: "en",
    targets: ["es", "fr", "de"],
  },
  files: {
    json: {
      include: ["locales/[locale].json"],
    },
  },
});
```

```json title="languine.json"
{
  "locale": {
    "source": "en",
    "targets": ["es", "fr", "de"]
  },
  "files": {
    "json": {
      "include": ["locales/[locale].json"]
    }
  }
}
```

ทั้งสองรูปแบบรองรับตัวเลือกการตั้งค่าเดียวกัน เลือก TypeScript เพื่อการตรวจสอบประเภทที่ดีกว่าและการสนับสนุน IDE หรือ JSON เพื่อความเรียบง่ายและความเข้ากันได้กับเครื่องมือที่กว้างขึ้น

มาดูรายละเอียดในแต่ละส่วนของไฟล์เหล่านี้กัน

---

## การตั้งค่าภาษา

ส่วน locale กำหนดภาษาที่ใช้ในการแปลของคุณ:

- source: ภาษาหลักของคุณสำหรับการสร้างเนื้อหา (เช่น "en" สำหรับภาษาอังกฤษ)
- targets: อาร์เรย์ของภาษาที่คุณต้องการแปลเป็น (เช่น ["es", "fr", "de"] สำหรับสเปน ฝรั่งเศส และเยอรมัน)

Languine รองรับรหัสภาษาที่หลากหลาย รวมถึง:
- รหัสง่าย: en, es, fr, de
- รหัสเฉพาะภูมิภาค: pt-BR, zh-CN, zh-TW

## การตั้งค่าไฟล์

ส่วนไฟล์กำหนดว่า Languine ควรแปลไฟล์ใด คุณสามารถกำหนดประเภทไฟล์และรูปแบบหลายประเภท:

---

### ไฟล์ JSON
```typescript
files: {
  json: {
    include: ["locales/[locale].json"],
  },
}
```

---

### ไฟล์ Markdown/MDX
```typescript
files: {
  md: {
    include: ["docs/[locale]/*.md"],
  },
  mdx: {
    include: ["blog/[locale]/*.mdx"],
  },
}
```

---

### ไฟล์แพลตฟอร์มมือถือ

สำหรับ iOS (Xcode):
```typescript
files: {
  "xcode-xcstrings": {
    include: ["Example/Localizable.xcstrings"],
  },
  // หรือสำหรับไฟล์ .strings แบบดั้งเดิม
  "xcode-strings": {
    include: ["Example/[locale].lproj/Localizable.strings"],
  },
}
```

---

สำหรับ Android:
```typescript
files: {
  android: {
    include: ["locales/[locale].xml"],
  },
}
```

## รูปแบบไฟล์

ตัวแทน [locale] ในรูปแบบไฟล์จะถูกแทนที่โดยอัตโนมัติด้วยรหัสภาษาที่เหมาะสมในระหว่างการแปล ตัวอย่างเช่น:

- locales/[locale].json จะกลายเป็น:
  - locales/en.json
  - locales/es.json
  - locales/fr.json

คุณสามารถรวมรูปแบบหลายรูปแบบสำหรับแต่ละประเภทไฟล์และใช้รูปแบบ glob:
```typescript
files: {
  json: {
    include: [
      "src/locales/[locale].json",
      "src/components/[locale]/*.json"
    ],
  },
}
```

---

## รหัสโครงการ (ไม่บังคับ)

หากคุณใช้บริการคลาวด์ของ Languine คุณสามารถรวมรหัสโครงการของคุณในไฟล์การตั้งค่า:

```typescript
export default defineConfig({
  projectId: "prj_your_project_id",
  locale: {
    // ... การตั้งค่าภาษา
  },
  files: {
    // ... การตั้งค่าไฟล์
  },
});
```

---

## การติดตามสถานะการแปล

Languine จะสร้างและรักษาไฟล์ languine.lock โดยอัตโนมัติเพื่อติดตามสถานะของการแปลของคุณ ไฟล์นี้:

- ติดตามว่าเนื้อหาใดถูกแปลแล้ว
- ช่วยระบุสิ่งที่ต้องอัปเดตเมื่อเนื้อหาต้นทางมีการเปลี่ยนแปลง
- ควรจะถูกบันทึกในระบบควบคุมเวอร์ชันของคุณ

คุณไม่จำเป็นต้องแก้ไขไฟล์ล็อคด้วยตนเอง - Languine จะจัดการมันโดยอัตโนมัติเมื่อคุณเรียกใช้คำสั่งการแปล

## แนวทางปฏิบัติที่ดีที่สุด

1. **การควบคุมเวอร์ชัน**: ควรบันทึกทั้ง languine.config.ts และ languine.lock ในที่เก็บของคุณเสมอ
2. **การจัดระเบียบไฟล์**: ควรรักษาไฟล์การแปลของคุณให้เป็นระเบียบในโครงสร้างไดเรกทอรีที่สอดคล้องกัน
3. **รหัสภาษา**: ใช้รหัสภาษามาตรฐานเพื่อให้แน่ใจว่ามีความเข้ากันได้
4. **รูปแบบ**: ใช้รูปแบบการรวมที่เฉพาะเจาะจงเพื่อหลีกเลี่ยงการแปลไฟล์ที่ไม่จำเป็น
5. **รหัสโครงการ**: รักษารหัสโครงการของคุณให้ปลอดภัยและไม่แชร์ต่อสาธารณะหากคุณใช้บริการคลาวด์
