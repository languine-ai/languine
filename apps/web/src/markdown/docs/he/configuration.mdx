# תצורה

לנגואין תומכת גם בקבצי תצורה מסוג TypeScript וגם בקבצי JSON כדי לנהל את הגדרות התרגום שלך. אתה יכול לבחור בין languine.config.ts עבור TypeScript או languine.json עבור פורמט JSON. שני הקבצים מגדירים את השפות המקוריות והיעד שלך, כמו גם את הקבצים שצריך לתרגם.

---

## תצורת בסיס

הנה דוגמה בסיסית בשני הפורמטים:

```typescript title="languine.config.ts"
import { defineConfig } from "languine";

export default defineConfig({
  locale: {
    source: "en",
    targets: ["es", "fr", "de"],
  },
  files: {
    json: {
      include: ["locales/[locale].json"],
    },
  },
});
```

```json title="languine.json"
{
  "locale": {
    "source": "en",
    "targets": ["es", "fr", "de"]
  },
  "files": {
    "json": {
      "include": ["locales/[locale].json"]
    }
  }
}
```

שני הפורמטים תומכים באותן אפשרויות תצורה. בחר ב-TypeScript עבור בדיקות סוגים טובות יותר ותמיכה ב-IDE, או ב-JSON עבור פשטות והתאמה רחבה יותר לכלים.

בואו נפרק את החלקים השונים של הקבצים הללו.

---

## תצורת שפה

חלק השפה מגדיר את שפות התרגום שלך:

- source: השפה הראשית שלך ליצירת תוכן (למשל, "en" לאנגלית)
- targets: מערך של שפות שאתה רוצה לתרגם אליהן (למשל, ["es", "fr", "de"] לספרדית, צרפתית וגרמנית)

לנגואין תומכת במגוון רחב של קודי שפה, כולל:
- קודים פשוטים: en, es, fr, de
- קודים ספציפיים לאזור: pt-BR, zh-CN, zh-TW

## תצורת קבצים

חלק הקבצים מגדיר אילו קבצים לנגואין צריכה לתרגם. אתה יכול להגדיר סוגי קבצים ודפוסים מרובים:

---

### קבצי JSON
```typescript
files: {
  json: {
    include: ["locales/[locale].json"],
  },
}
```

---

### קבצי Markdown/MDX
```typescript
files: {
  md: {
    include: ["docs/[locale]/*.md"],
  },
  mdx: {
    include: ["blog/[locale]/*.mdx"],
  },
}
```

---

### קבצי פלטפורמת מובייל

עבור iOS (Xcode):
```typescript
files: {
  "xcode-xcstrings": {
    include: ["Example/Localizable.xcstrings"],
  },
  // או עבור קבצי .strings מסורתיים
  "xcode-strings": {
    include: ["Example/[locale].lproj/Localizable.strings"],
  },
}
```

---

עבור Android:
```typescript
files: {
  android: {
    include: ["locales/[locale].xml"],
  },
}
```

## דפוסי קבצים

המקום המיועד [locale] בדפוסי הקבצים מוחלף אוטומטית בקוד השפה המתאים במהלך התרגום. לדוגמה:

- locales/[locale].json הופך ל:
  - locales/en.json
  - locales/es.json
  - locales/fr.json

אתה יכול לכלול דפוסים מרובים עבור כל סוג קובץ ולהשתמש בדפוסי glob:
```typescript
files: {
  json: {
    include: [
      "src/locales/[locale].json",
      "src/components/[locale]/*.json"
    ],
  },
}
```

## מזהה פרויקט (אופציונלי)

אם אתה משתמש בשירותי הענן של לנגואין, אתה יכול לכלול את מזהה הפרויקט שלך בתצורה:

```typescript
export default defineConfig({
  projectId: "prj_your_project_id",
  locale: {
    // ... תצורת שפה
  },
  files: {
    // ... תצורת קבצים
  },
});
```

---

## מעקב אחר מצב התרגום

לנגואין יוצרת ומתחזקת אוטומטית קובץ languine.lock כדי לעקוב אחר מצב התרגומים שלך. קובץ זה:

- עוקב אחר איזה תוכן תורגם
- עוזר לזהות מה צריך להתעדכן כאשר תוכן המקור משתנה
- יש להתחייב למערכת בקרת הגרסאות שלך

אתה לא צריך לערוך ידנית את קובץ הנעילה - לנגואין מנהלת אותו אוטומטית כאשר אתה מריץ פקודות תרגום.

## שיטות עבודה מומלצות

1. **בקרת גרסאות**: תמיד התחייב גם ל-languine.config.ts וגם ל-languine.lock למאגר שלך
2. **ארגון קבצים**: שמור על קבצי התרגום שלך מאורגנים במבנה תיקיות עקבי
3. **קודי שפה**: השתמש בקודי שפה סטנדרטיים כדי להבטיח תאימות
4. **דפוסים**: השתמש בדפוסי הכללה ספציפיים כדי להימנע מתרגום קבצים לא נחוצים
5. **מזהה פרויקט**: שמור על מזהה הפרויקט שלך מאובטח ואל תשתף אותו בפומבי אם אתה משתמש בשירותי ענן
