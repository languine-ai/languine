# פעולות GitHub

עם פעולת GitHub, אתה יכול בקלות לתרגם את הקוד שלך על ידי דחיפת הקוד שלך למאגר, פעולת GitHub תתרגם את המפתחות החדשים שלך, תיצור בקשת משיכה או תתחייב את השינויים שלך.
זה חוסך לך הרבה זמן ומאמץ תוך שמירה על תרגומים באיכות גבוהה.

---

## התקנה

כדי להשתמש בפעולת GitHub של Languine במאגר שלך, תצטרך:

1. להגדיר את הסודות של המאגר שלך עם האישורים שלך ל-Languine
2. ליצור קובץ זרימת עבודה במאגר שלך

### הגדרת סודות המאגר

הוסף את הסודות הבאים בהגדרות המאגר שלך (הגדרות > סודות ומשתנים > פעולות):

- `LANGUINE_API_KEY`: מפתח ה-API שלך ל-Languine
- `LANGUINE_PROJECT_ID`: מזהה הפרויקט שלך ב-Languine

אתה יכול למצוא ערכים אלה בלוח המחוונים של Languine תחת הגדרות הפרויקט.



### משתני סביבה

פעולת GitHub משתמשת במשתני הסביבה הבאים:
- `LANGUINE_API_KEY` (נדרש): מפתח ה-API שלך ל-Languine לצורך אימות
- `LANGUINE_PROJECT_ID` (נדרש): מזהה הפרויקט שלך ב-Languine 
- `GITHUB_TOKEN` (נדרש): מסופק אוטומטית על ידי GitHub, משמש ליצירת בקשות משיכה

### דוגמת זרימת עבודה

צור קובץ חדש .github/workflows/languine.yml במאגר שלך עם התוכן הבא:

```ts title=".github/workflows/release.yml"
name: פריסת אתר

on:
  push:
    branches:
      - '**'
    paths:
      - apps/web/**

jobs:
  translate:
    runs-on: ubuntu-latest
    permissions:
      contents: write
      pull-requests: write
    steps:
      - uses: actions/checkout@v4
      
      - name: פעולת Languine
        uses: languine/languine-action@v1
        env:
           GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
        with:
          api-key: ${{ secrets.LANGUINE_API_KEY }}
          project-id: ${{ secrets.LANGUINE_PROJECT_ID }}
```
 
 זרימת עבודה זו ת:
- תרוץ על דחיפות לענף הראשי
- תרוץ על בקשות משיכה לענף הראשי
- ניתן להפעיל ידנית מהכרטיסייה פעולות
- תבצע את הקוד שלך ותפעיל את תהליך התרגום של Languine
- תיצור בקשת משיכה עם כל התרגומים החדשים (ניתן להגדיר)

#### אפשרות ספריית עבודה

אפשרות `working-directory` מאפשרת לך לציין היכן ממוקמים קבצי התרגום שלך. זה שימושי במיוחד במונורפוז שבהם התרגומים שלך עשויים להיות בחבילה או בתיקיית אפליקציה ספציפית. לדוגמה:

#### אפשרות בקשת משיכה

אפשרות `create-pull-request` שולטת האם הפעולה יוצרת בקשת משיכה עם השינויים או מתחייבת ישירות לענף:

```typescript title=".github/workflows/languine.yml"
with:
  create-pull-request: true # צור בקשת משיכה עם השינויים
```

כאשר מוגדר ל-`true`, הפעולה ת:
- תיצור ענף חדש עם השינויים
- תיצור בקשת משיכה לסקירה
- תוסיף תוויות ותיאור מתאימים

כאשר מוגדר ל-`false` (ברירת מחדל), הפעולה ת:
- תתחייב את השינויים ישירות לענף הנוכחי
- תדחוף את השינויים מבלי ליצור בקשת משיכה
