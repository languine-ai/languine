# Konfigurointi

Languine tukee sekä TypeScript- että JSON-konfiguraatiotiedostoja käännöksesi hallintaan. Voit valita joko languine.config.ts TypeScriptille tai languine.json JSON-muodolle. Molemmat tiedostot määrittelevät lähde- ja kohdekielet sekä käännettävät tiedostot.

---

## Peruskonfigurointi

Tässä on perusesimerkki molemmissa muodoissa:

```typescript title="languine.config.ts"
import { defineConfig } from "languine";

export default defineConfig({
  locale: {
    source: "en",
    targets: ["es", "fr", "de"],
  },
  files: {
    json: {
      include: ["locales/[locale].json"],
    },
  },
});
```

```json title="languine.json"
{
  "locale": {
    "source": "en",
    "targets": ["es", "fr", "de"]
  },
  "files": {
    "json": {
      "include": ["locales/[locale].json"]
    }
  }
}
```

Molemmat muodot tukevat samoja konfigurointivaihtoehtoja. Valitse TypeScript paremman tyyppitarkistuksen ja IDE-tuen vuoksi tai JSON yksinkertaisuuden ja laajemman työkaluyhteensopivuuden vuoksi.

Käydään läpi näiden tiedostojen eri osiot.

---

## Paikalliskonfigurointi

Paikallisosio määrittelee käännöskielet:

- source: Pääasiallinen kielesi sisällön luomiseen (esim. "en" englanniksi)
- targets: Taulukko kielistä, joihin haluat kääntää (esim. ["es", "fr", "de"] espanjaksi, ranskaksi ja saksaksi)

Languine tukee laajaa valikoimaa kielikoodeja, mukaan lukien:
- Yksinkertaiset koodit: en, es, fr, de
- Aluekohtaiset koodit: pt-BR, zh-CN, zh-TW

## Tiedostokonfigurointi

Tiedostot-osio määrittelee, mitkä tiedostot Languinen tulisi kääntää. Voit konfiguroida useita tiedostotyyppejä ja kuvioita:

---

### JSON-tiedostot
```typescript
files: {
  json: {
    include: ["locales/[locale].json"],
  },
}
```

---

### Markdown/MDX-tiedostot
```typescript
files: {
  md: {
    include: ["docs/[locale]/*.md"],
  },
  mdx: {
    include: ["blog/[locale]/*.mdx"],
  },
}
```

---

### Mobiilialustatiedostot

iOS (Xcode):
```typescript
files: {
  "xcode-xcstrings": {
    include: ["Example/Localizable.xcstrings"],
  },
  // Tai perinteisille .strings-tiedostoille
  "xcode-strings": {
    include: ["Example/[locale].lproj/Localizable.strings"],
  },
}
```

---

Androidille:
```typescript
files: {
  android: {
    include: ["locales/[locale].xml"],
  },
}
```

## Tiedostokuvio

Tiedostokuvioissa [locale]-paikkamerkki korvataan automaattisesti oikealla kielikoodilla käännöksen aikana. Esimerkiksi:

- locales/[locale].json muuttuu:
  - locales/en.json
  - locales/es.json
  - locales/fr.json

Voit sisällyttää useita kuvioita jokaiselle tiedostotyypille ja käyttää glob-kuvioita:
```typescript
files: {
  json: {
    include: [
      "src/locales/[locale].json",
      "src/components/[locale]/*.json"
    ],
  },
}
```

## Projektitunnus (valinnainen)

Jos käytät Languinen pilvipalveluja, voit sisällyttää projektitunnuksesi konfiguraatioon:

```typescript
export default defineConfig({
  projectId: "prj_your_project_id",
  locale: {
    // ... paikalliskonfigurointi
  },
  files: {
    // ... tiedostokonfigurointi
  },
});
```

## Käännöksen tilan seuranta

Languine luo ja ylläpitää automaattisesti languine.lock-tiedostoa käännöksesi tilan seuraamiseksi. Tämä tiedosto:

- Seuraa, mikä sisältö on käännetty
- Auttaa tunnistamaan, mitä on päivitettävä, kun lähdesisältö muuttuu
- Tulisi sitoa versionhallintajärjestelmääsi

Sinun ei tarvitse muokata lukitustiedostoa käsin - Languine hallitsee sitä automaattisesti, kun suoritat käännöskomentoja.

## Parhaat käytännöt

1. **Versionhallinta**: Sitoudu aina sekä languine.config.ts että languine.lock -tiedostot varastoon
2. **Tiedostojen järjestäminen**: Pidä käännöstiedostosi järjestettyinä johdonmukaisessa hakemistorakenteessa
3. **Kielikoodit**: Käytä standardeja kielikoodeja yhteensopivuuden varmistamiseksi
4. **Kuvioita**: Käytä erityisiä sisällytyskuvioita välttääksesi tarpeettomien tiedostojen kääntämisen
5. **Projektitunnus**: Pidä projektitunnuksesi turvassa äläkä jaa sitä julkisesti, jos käytät pilvipalveluja
