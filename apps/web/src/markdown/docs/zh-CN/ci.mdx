# CI/CD

Languine 提供 CI/CD 集成，帮助您翻译应用程序。

---

## 工作原理

Languine 中的 CI/CD 工作流程遵循结构化过程，以自动化应用程序中的翻译更新：

### 1. 动作初始化
- 当触发动作时，工作流程开始
- 设置必要的 Git 配置和凭据

### 2. Languine 引擎处理
- Languine 引擎处理您应用程序的内容
- 使用配置的设置处理翻译任务

### 3. 变更检测
系统检查是否有任何翻译变更，并遵循以下两条路径之一：

**当发现变更时：**
- 变更被提交到代码库
- 变更被推送到指定分支
- 创建拉取请求（PR）

**当未发现变更时：**
- 工作流程在不创建 PR 的情况下完成

### 4. 拉取请求创建
当检测到变更时，创建一个拉取请求，内容包括：

**PR 内容：**
- 清晰的标题，指示翻译更新
- 变更的详细描述
- 质量检查确认
- 审核说明
- 文档和支持链接

**PR 描述包括：**
- 翻译更新的摘要
- 变更列表（更新的翻译、术语一致性）
- 审核者的后续步骤
- 文档和支持资源的链接

该工作流程旨在实现自动化，同时仍允许在变更合并到您的代码库之前进行人工审核和质量控制。它遵循版本控制最佳实践，并确保翻译通过 Git 历史记录得到妥善跟踪和记录。
