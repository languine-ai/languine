# React Email

Languine stöder React Email-mallar, vilket gör att du kan hantera översättningar för dina e-postmallar byggda med React Email. Denna integration hanterar både textinnehåll och HTML-attribut samtidigt som den bevarar React-komponentstrukturen.

---

## Inställning

Först, se till att du har en `languine.config.json` eller `languine.config.ts` konfigurationsfil i rotmappen av ditt projekt. Här är ett exempel:

```json
{
  "locale": {
    "source": "en",
    "targets": ["sv", "de", "fr"]
  },
  "files": {
    "react-email": {
      "include": ["emails/[locale]/*.tsx"]
    }
  }
}
```

## Översättning

Med din konfiguration inställd, kör:

```bash
npx languine@latest translate
```

När du kör detta kommando kommer Languine att:

- Ladda dina källor React Email-mallar (t.ex. emails/en/*.tsx)
- Upptäcka eventuella nya eller ändrade översättningssträngar
- Generera översättningar för dina målspråk
- Skapa eller uppdatera e-postmallar för målspråket (t.ex. emails/sv/*.tsx)
- Bevara React-komponentstruktur och props
- Bibehålla HTML-formattering och stil
- Hantera interpolering och dynamiskt innehåll
