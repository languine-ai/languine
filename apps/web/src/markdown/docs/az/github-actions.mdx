# GitHub Actions

GitHub action ilə kodunuzu sadəcə repositoriyaya yükləməklə asanlıqla tərcümə edə bilərsiniz, GitHub action yeni açarları tərcümə edəcək, ya bir pull request yaradacaq, ya da dəyişikliklərinizi təsdiqləyəcək.
Bu, sizə çox vaxt və əziyyət qazandırır, eyni zamanda yüksək keyfiyyətli tərcümələri qoruyur.

---

## Quraşdırma

Repositoriyanızda Languine GitHub Action-dan istifadə etmək üçün:

1. Languine məlumatlarınızla repositoriya gizli məlumatlarını qurun
2. Repositoriyanızda bir iş axını faylı yaradın

### Repositoriya Gizli Məlumatlarını Quraşdırma

Repositoriya parametrlərinizdə (Parametrlər > Gizli və dəyişənlər > Actions) aşağıdakı gizli məlumatları əlavə edin:

- `LANGUINE_API_KEY`: Sizin Languine API açarınız
- `LANGUINE_PROJECT_ID`: Sizin Languine layihə ID-niz

Bu dəyərləri Languine idarəetmə panelinizdə Layihə Parametrləri altında tapa bilərsiniz.



### Mühit Dəyişənləri

GitHub Action aşağıdakı mühit dəyişənlərindən istifadə edir:
- `LANGUINE_API_KEY` (Tələb olunur): Doğrulama üçün sizin Languine API açarınız
- `LANGUINE_PROJECT_ID` (Tələb olunur): Sizin Languine layihə ID-niz 
- `GITHUB_TOKEN` (Tələb olunur): GitHub tərəfindən avtomatik təmin edilir, PR-lar yaratmaq üçün istifadə olunur

### Nümunə İş Axını

Repositoriyanızda aşağıdakı məzmunla .github/workflows/languine.yml adlı yeni bir fayl yaradın:

```ts title=".github/workflows/release.yml"
name: Vebsaytı Yerləşdir

on:
  push:
    branches:
      - '**'
    paths:
      - apps/web/**

jobs:
  translate:
    runs-on: ubuntu-latest
    permissions:
      contents: write
      pull-requests: write
    steps:
      - uses: actions/checkout@v4
      
      - name: Languine Action
        uses: languine/languine-action@v1
        env:
           GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
        with:
          api-key: ${{ secrets.LANGUINE_API_KEY }}
          project-id: ${{ secrets.LANGUINE_PROJECT_ID }}
```
 
 Bu iş axını:
- Əsas budağa yüklənmələrdə işləyəcək
- Əsas budağa pull request-lərdə işləyəcək
- Actions tabından əl ilə tetiklenebilir
- Kodunuzu checkout edəcək və Languine tərcümə prosesini icra edəcək
- Hər hansı yeni tərcümələri olan bir pull request yaradacaq (konfiqurasiya edilə bilər)

#### İşçi Qovluq Seçimi

`working-directory` seçimi tərcümə fayllarınızın harada yerləşdiyini göstərməyə imkan verir. Bu, tərcümələrinizin müəyyən bir paket və ya tətbiq qovluğunda ola biləcəyi monoreposlar üçün xüsusilə faydalıdır. Məsələn:

#### Pull Request Seçimi

`create-pull-request` seçimi, action-un dəyişikliklərlə bir pull request yaradıb-yaratmayacağını və ya onları birbaşa budağa təsdiqləyib-təsdiqləməyəcəyini idarə edir:

```typescript title=".github/workflows/languine.yml"
with:
  create-pull-request: true # dəyişikliklərlə bir pull request yaradın
```

`true` olaraq təyin edildikdə, action:
- Dəyişikliklərlə yeni bir budaq yaradacaq
- İcmal üçün bir pull request yaradacaq
- Müvafiq etiketlər və təsvir əlavə edəcək

`false` (standart) olaraq təyin edildikdə, action:
- Dəyişiklikləri birbaşa cari budağa təsdiqləyəcək
- Pull request yaratmadan dəyişiklikləri yükləyəcək
