# CLI

Languine fornece uma CLI para ajudar você a traduzir sua aplicação.

## Visão Geral dos Comandos

### translate

O comando `translate` é usado para traduzir sua aplicação.

```bash title="Terminal"
# Iniciar a tradução da sua aplicação
languine translate

# Opções:
#   --source-locale    Especificar o locale de origem (padrão: en)
#   --target-locale    Especificar o(s) locale(s) de destino para traduzir
#   --config          Caminho para seu arquivo languine.config.js
```

### init

```bash title="Terminal"
# Inicializar um novo projeto Languine
languine init

# Isso irá:
# - Criar um arquivo languine.config.js
# - Configurar a estrutura do seu projeto
# - Inicializar diretórios necessários
```

### status

```bash title="Terminal"
# Verificar o status das suas traduções
languine status

# Mostra:
# - Progresso atual da tradução
# - Traduções faltantes
# - Estatísticas de tradução
```

### validate

```bash title="Terminal"
# Validar seus arquivos de tradução
languine validate

# Verifica:
# - Estrutura do arquivo
# - Chaves faltantes
# - Consistência de formato
```

### export

```bash title="Terminal"
# Exportar suas traduções
languine export

# Opções:
#   --format          Formato de exportação (json, yaml, etc.)
#   --output         Diretório de saída
```

### import

```bash title="Terminal"
# Importar traduções de arquivos externos
languine import

# Opções:
#   --source         Arquivo ou diretório de origem
#   --format         Formato de importação (json, yaml, etc.)
```

## Opções Globais

Todos os comandos suportam as seguintes opções globais:

```bash title="Terminal"
# Opções globais disponíveis para todos os comandos
languine [comando] [opções]

# Opções comuns:
#   --help, -h       Mostrar informações de ajuda
#   --version, -v    Mostrar informações da versão
#   --verbose        Ativar registro detalhado
#   --config         Especificar o caminho do arquivo de configuração
```
