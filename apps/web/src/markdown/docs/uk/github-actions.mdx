# GitHub Actions

За допомогою дії GitHub ви можете легко перекладати свій код, просто завантажуючи його в репозиторій, після чого дія GitHub перекладе ваші нові ключі, створить запит на злиття або зафіксує ваші зміни.
Це економить вам багато часу та зусиль, зберігаючи при цьому високу якість перекладів.

---

## Встановлення

Щоб використовувати GitHub Action Languine у вашому репозиторії, вам потрібно:

1. Налаштувати секрети репозиторію з вашими обліковими даними Languine
2. Створити файл робочого процесу у вашому репозиторії

### Налаштування секретів репозиторію

Додайте наступні секрети в налаштуваннях вашого репозиторію (Налаштування > Секрети та змінні > Дії):

- `LANGUINE_API_KEY`: Ваш ключ API Languine
- `LANGUINE_PROJECT_ID`: Ваш ідентифікатор проекту Languine

Ви можете знайти ці значення на панелі управління Languine у розділі Налаштування проекту.



### Змінні середовища

GitHub Action використовує наступні змінні середовища:
- `LANGUINE_API_KEY` (Обов'язково): Ваш ключ API Languine для автентифікації
- `LANGUINE_PROJECT_ID` (Обов'язково): Ваш ідентифікатор проекту Languine 
- `GITHUB_TOKEN` (Обов'язково): Автоматично надається GitHub, використовується для створення запитів на злиття

### Приклад робочого процесу

Створіть новий файл .github/workflows/languine.yml у вашому репозиторії з наступним вмістом:

```ts title=".github/workflows/release.yml"
name: Розгортання вебсайту

on:
  push:
    branches:
      - '**'
    paths:
      - apps/web/**

jobs:
  translate:
    runs-on: ubuntu-latest
    permissions:
      contents: write
      pull-requests: write
    steps:
      - uses: actions/checkout@v4
      
      - name: Дія Languine
        uses: languine/languine-action@v1
        env:
           GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
        with:
          api-key: ${{ secrets.LANGUINE_API_KEY }}
          project-id: ${{ secrets.LANGUINE_PROJECT_ID }}
```
 
 Цей робочий процес:
- Виконується при завантаженнях у основну гілку
- Виконується при запитах на злиття до основної гілки
- Може бути запущений вручну з вкладки Дії
- Витягує ваш код і запускає процес перекладу Languine
- Створює запит на злиття з будь-якими новими перекладами (можна налаштувати)

#### Опція запиту на злиття

Опція `create-pull-request` контролює, чи створює дія запит на злиття зі змінами або безпосередньо фіксує їх у гілці:

```typescript title=".github/workflows/languine.yml"
with:
  create-pull-request: true # створити запит на злиття зі змінами
```

Коли встановлено на `true`, дія:
- Створить нову гілку зі змінами
- Створить запит на злиття для перегляду
- Додасть відповідні мітки та опис

Коли встановлено на `false` (за замовчуванням), дія:
- Зафіксує зміни безпосередньо в поточній гілці
- Відправить зміни без створення запиту на злиття
