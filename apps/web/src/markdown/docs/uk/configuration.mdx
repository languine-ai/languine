# Налаштування

Languine підтримує як файли конфігурації TypeScript, так і JSON для управління вашими налаштуваннями перекладу. Ви можете вибрати languine.config.ts для TypeScript або languine.json для формату JSON. Обидва файли визначають ваші вихідні та цільові мови, а також файли, які потрібно перекласти.

---

## Основне Налаштування

Ось базовий приклад в обох форматах:

```typescript title="languine.config.ts"
import { defineConfig } from "languine";

export default defineConfig({
  locale: {
    source: "en",
    targets: ["es", "fr", "de"],
  },
  files: {
    json: {
      include: ["locales/[locale].json"],
    },
  },
});
```

```json title="languine.json"
{
  "locale": {
    "source": "en",
    "targets": ["es", "fr", "de"]
  },
  "files": {
    "json": {
      "include": ["locales/[locale].json"]
    }
  }
}
```

Обидва формати підтримують однакові параметри конфігурації. Виберіть TypeScript для кращої перевірки типів та підтримки IDE, або JSON для простоти та ширшої сумісності з інструментами.

Давайте розглянемо різні секції цих файлів.

---

## Налаштування Локалі

Секція локалі визначає ваші мови перекладу:

- source: Ваша основна мова для створення контенту (наприклад, "en" для англійської)
- targets: Масив мов, на які ви хочете перекласти (наприклад, ["es", "fr", "de"] для іспанської, французької та німецької)

Languine підтримує широкий спектр кодів мов, включаючи:
- Простi коди: en, es, fr, de
- Регіональні коди: pt-BR, zh-CN, zh-TW

## Налаштування Файлів

Секція файлів визначає, які файли Languine має перекласти. Ви можете налаштувати кілька типів файлів та шаблонів:

---

### JSON Файли
```typescript
files: {
  json: {
    include: ["locales/[locale].json"],
  },
}
```

---

### Markdown/MDX Файли
```typescript
files: {
  md: {
    include: ["docs/[locale]/*.md"],
  },
  mdx: {
    include: ["blog/[locale]/*.mdx"],
  },
}
```

---

### Файли Мобільних Платформ

Для iOS (Xcode):
```typescript
files: {
  "xcode-xcstrings": {
    include: ["Example/Localizable.xcstrings"],
  },
  // Або для традиційних .strings файлів
  "xcode-strings": {
    include: ["Example/[locale].lproj/Localizable.strings"],
  },
}
```

---

Для Android:
```typescript
files: {
  android: {
    include: ["locales/[locale].xml"],
  },
}
```

## Шаблони Файлів

Плейсхолдер [locale] у шаблонах файлів автоматично замінюється на відповідний код мови під час перекладу. Наприклад:

- locales/[locale].json стає:
  - locales/en.json
  - locales/es.json
  - locales/fr.json

Ви можете включити кілька шаблонів для кожного типу файлу та використовувати шаблони glob:
```typescript
files: {
  json: {
    include: [
      "src/locales/[locale].json",
      "src/components/[locale]/*.json"
    ],
  },
}
```

## Ідентифікатор Проекту (Необов'язково)

Якщо ви використовуєте хмарні сервіси Languine, ви можете включити свій ідентифікатор проекту в конфігурацію:

```typescript
export default defineConfig({
  projectId: "prj_your_project_id",
  locale: {
    // ... конфігурація локалі
  },
  files: {
    // ... конфігурація файлів
  },
});
```

---

## Відстеження Стану Перекладу

Languine автоматично створює та підтримує файл languine.lock для відстеження стану ваших перекладів. Цей файл:

- Відстежує, який контент був перекладений
- Допомагає визначити, що потрібно оновити, коли змінюється вихідний контент
- Має бути зафіксований у вашій системі контролю версій

Вам не потрібно вручну редагувати файл блокування - Languine управляє ним автоматично, коли ви виконуєте команди перекладу.

## Найкращі Практики

1. **Контроль Версій**: Завжди фіксуйте як languine.config.ts, так і languine.lock у вашому репозиторії
2. **Організація Файлів**: Зберігайте свої файли перекладу організованими в послідовній структурі каталогів
3. **Коди Мов**: Використовуйте стандартні коди мов для забезпечення сумісності
4. **Шаблони**: Використовуйте специфічні шаблони включення, щоб уникнути перекладу непотрібних файлів
5. **Ідентифікатор Проекту**: Зберігайте свій ідентифікатор проекту в безпеці та не діліться ним публічно, якщо ви використовуєте хмарні сервіси
