# CLI

Languine надає CLI для допомоги у перекладі вашого додатку.

## Огляд команд

### translate

Команда `translate` використовується для перекладу вашого додатку.

```bash title="Terminal"
# Розпочати переклад вашого додатку
languine translate

# Опції:
#   --source-locale    Вкажіть вихідну локалізацію (за замовчуванням: en)
#   --target-locale    Вкажіть цільову локалізацію(ї) для перекладу
#   --config          Шлях до вашого файлу languine.config.js
```

### init

```bash title="Terminal"
# Ініціалізувати новий проект Languine
languine init

# Це зробить:
# - Створить файл languine.config.js
# - Налаштує структуру вашого проекту
# - Ініціалізує необхідні каталоги
```

### status

```bash title="Terminal"
# Перевірити статус ваших перекладів
languine status

# Показує:
# - Поточний прогрес перекладу
# - Відсутні переклади
# - Статистику перекладу
```

### validate

```bash title="Terminal"
# Перевірити ваші файли перекладу
languine validate

# Перевіряє:
# - Структуру файлів
# - Відсутні ключі
# - Узгодженість формату
```

### export

```bash title="Terminal"
# Експортувати ваші переклади
languine export

# Опції:
#   --format          Формат експорту (json, yaml тощо)
#   --output         Каталог виводу
```

### import

```bash title="Terminal"
# Імпортувати переклади з зовнішніх файлів
languine import

# Опції:
#   --source         Вихідний файл або каталог
#   --format         Формат імпорту (json, yaml тощо)
```

## Глобальні опції

Усі команди підтримують такі глобальні опції:

```bash title="Terminal"
# Глобальні опції, доступні для всіх команд
languine [command] [options]

# Загальні опції:
#   --help, -h       Показати інформацію про допомогу
#   --version, -v    Показати інформацію про версію
#   --verbose        Увімкнути детальне ведення журналу
#   --config         Вказати шлях до файлу конфігурації
```
