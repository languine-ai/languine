# CI/CD

Languine надає інтеграцію CI/CD, щоб допомогти вам перекладати ваш додаток.

---

## Як це працює

Процес CI/CD в Languine слідує структурованому процесу для автоматизації оновлень перекладу у вашому додатку:

### 1. Ініціалізація дії
- Процес починається, коли ініціюється дія
- Налаштовує необхідну конфігурацію Git та облікові дані

### 2. Обробка двигуном Languine
- Двигун Languine обробляє вміст вашого додатку
- Виконує завдання перекладу, використовуючи налаштовані параметри

### 3. Виявлення змін
Система перевіряє наявність змін у перекладі та слідує одному з двох шляхів:

**Коли знайдено зміни:**
- Зміни фіксуються в репозиторії
- Зміни відправляються в зазначену гілку
- Створюється запит на злиття (PR)

**Коли зміни не знайдено:**
- Процес завершується без створення PR

### 4. Створення запиту на злиття
Коли виявлено зміни, створюється запит на злиття з:

**Змістом PR:**
- Чіткий заголовок, що вказує на оновлення перекладу
- Детальний опис змін
- Підтвердження перевірки якості
- Інструкції для рецензування
- Посилання на документацію та підтримку

**Опис PR включає:**
- Резюме оновлень перекладу
- Список змін (оновлені переклади, узгодженість термінології)
- Наступні кроки для рецензентів
- Посилання на документацію та ресурси підтримки
