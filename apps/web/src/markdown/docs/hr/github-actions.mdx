# GitHub Actions

Uz GitHub akciju, možete lako prevesti svoj kod jednostavnim slanjem svog koda u repozitorij, GitHub akcija će zatim prevesti vaše nove ključeve, bilo će stvorena pull zahtjev ili će se promjene potvrditi.
To vam štedi puno vremena i truda dok održavate visoku kvalitetu prijevoda.

---

## Instalacija

Da biste koristili Languine GitHub akciju u svom repozitoriju, trebate:

1. Postaviti tajne repozitorija s vašim Languine vjerodajnicama
2. Stvoriti datoteku radnog toka u svom repozitoriju

### Postavljanje Tajni Repozitorija

Dodajte sljedeće tajne u postavkama vašeg repozitorija (Postavke > Tajne i varijable > Akcije):

- `LANGUINE_API_KEY`: Vaš Languine API ključ
- `LANGUINE_PROJECT_ID`: Vaš Languine ID projekta

Ove vrijednosti možete pronaći na svom Languine nadzornoj ploči pod Postavke projekta.



### Varijable Okruženja

GitHub akcija koristi sljedeće varijable okruženja:
- `LANGUINE_API_KEY` (Obavezno): Vaš Languine API ključ za autentifikaciju
- `LANGUINE_PROJECT_ID` (Obavezno): Vaš Languine ID projekta 
- `GITHUB_TOKEN` (Obavezno): Automatski osiguran od strane GitHub-a, koristi se za stvaranje PR-ova

### Primjer Radnog Toka

Stvorite novu datoteku .github/workflows/languine.yml u svom repozitoriju s sljedećim sadržajem:

```ts title=".github/workflows/release.yml"
name: Deploy Website

on:
  push:
    branches:
      - '**'
    paths:
      - apps/web/**

jobs:
  translate:
    runs-on: ubuntu-latest
    permissions:
      contents: write
      pull-requests: write
    steps:
      - uses: actions/checkout@v4
      
      - name: Languine Action
        uses: languine/languine-action@v1
        env:
           GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
        with:
          api-key: ${{ secrets.LANGUINE_API_KEY }}
          project-id: ${{ secrets.LANGUINE_PROJECT_ID }}
```
 
 Ovaj radni tok će:
- Raditi na slanju na glavnu granu
- Raditi na pull zahtjevima na glavnu granu
- Može se pokrenuti ručno s kartice Akcije
- Preuzeti vaš kod i pokrenuti Languine proces prevođenja
- Stvoriti pull zahtjev s bilo kojim novim prijevodima (konfigurabilno)

#### Opcija Pull Zahtjeva

Opcija `create-pull-request` kontrolira hoće li akcija stvoriti pull zahtjev s promjenama ili ih izravno potvrditi u granu:

```typescript title=".github/workflows/languine.yml"
with:
  create-pull-request: true # stvoriti pull zahtjev s promjenama
```

Kada je postavljeno na `true`, akcija će:
- Stvoriti novu granu s promjenama
- Stvoriti pull zahtjev za pregled
- Dodati odgovarajuće oznake i opis

Kada je postavljeno na `false` (zadano), akcija će:
- Potvrditi promjene izravno u trenutnu granu
- Pritisnuti promjene bez stvaranja pull zahtjeva
