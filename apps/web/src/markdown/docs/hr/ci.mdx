# CI/CD

Languine pruža CI/CD integraciju koja vam pomaže u prevođenju vaše aplikacije.

---

## Kako to Funkcionira

CI/CD radni tok u Languineu slijedi strukturirani proces za automatizaciju ažuriranja prijevoda u vašoj aplikaciji:

### 1. Inicijalizacija Akcije
- Radni tok započinje kada se akcija pokrene
- Postavlja potrebne Git konfiguracije i vjerodajnice

### 2. Obrada Languine Motora
- Languine motor obrađuje sadržaj vaše aplikacije
- Rješava zadatke prevođenja koristeći konfigurirane postavke

### 3. Otkrivanje Promjena
Sustav provjerava ima li promjena u prijevodu i slijedi jedan od dva puta:

**Kada se Pronađu Promjene:**
- Promjene se potvrđuju u repozitoriju
- Promjene se šalju na određenu granu
- Kreira se Pull Request (PR)

**Kada Nema Pronađenih Promjena:**
- Radni tok se završava bez kreiranja PR-a

### 4. Kreiranje Pull Request-a
Kada se otkriju promjene, kreira se Pull Request s:

**Sadržajem PR-a:**
- Jasan naslov koji označava ažuriranja prijevoda
- Detaljan opis promjena
- Potvrde o provjeri kvalitete
- Upute za pregled
- Linkovi na dokumentaciju i podršku

**Opis PR-a Uključuje:**
- Sažetak ažuriranja prijevoda
- Popis onoga što je promijenjeno (ažurirani prijevodi, dosljednost terminologije)
- Sljedeće korake za recenzente
- Linkove na dokumentaciju i resurse podrške
