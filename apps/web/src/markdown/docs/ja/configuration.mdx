# 設定

Languineは、翻訳設定を管理するためにTypeScriptとJSONの両方の設定ファイルをサポートしています。TypeScript用のlanguine.config.tsまたはJSON形式のlanguine.jsonのいずれかを選択できます。両方のファイルは、ソース言語とターゲット言語、および翻訳が必要なファイルを定義します。

---

## 基本設定

以下は、両方の形式の基本的な例です：

```typescript title="languine.config.ts"
import { defineConfig } from "languine";

export default defineConfig({
  locale: {
    source: "en",
    targets: ["es", "fr", "de"],
  },
  files: {
    json: {
      include: ["locales/[locale].json"],
    },
  },
});
```

```json title="languine.json"
{
  "locale": {
    "source": "en",
    "targets": ["es", "fr", "de"]
  },
  "files": {
    "json": {
      "include": ["locales/[locale].json"]
    }
  }
}
```

両方の形式は、同じ設定オプションをサポートしています。より良い型チェックとIDEサポートのためにTypeScriptを選択するか、シンプルさと広範なツール互換性のためにJSONを選択してください。

これらのファイルの異なるセクションを分解してみましょう。

---

## ロケール設定

ロケールセクションは、翻訳言語を定義します：

- source: コンテンツ作成のための主要言語（例："en"は英語）
- targets: 翻訳したい言語の配列（例：["es", "fr", "de"]はスペイン語、フランス語、ドイツ語）

Languineは、以下を含む幅広い言語コードをサポートしています：
- シンプルなコード: en, es, fr, de
- 地域特有のコード: pt-BR, zh-CN, zh-TW

## ファイル設定

ファイルセクションは、Languineが翻訳すべきファイルを定義します。複数のファイルタイプとパターンを設定できます：

---

### JSONファイル
```typescript
files: {
  json: {
    include: ["locales/[locale].json"],
  },
}
```

---

### Markdown/MDXファイル
```typescript
files: {
  md: {
    include: ["docs/[locale]/*.md"],
  },
  mdx: {
    include: ["blog/[locale]/*.mdx"],
  },
}
```

---

### モバイルプラットフォームファイル

iOS（Xcode）の場合：
```typescript
files: {
  "xcode-xcstrings": {
    include: ["Example/Localizable.xcstrings"],
  },
  // 伝統的な.stringsファイルの場合
  "xcode-strings": {
    include: ["Example/[locale].lproj/Localizable.strings"],
  },
}
```

---

Androidの場合：
```typescript
files: {
  android: {
    include: ["locales/[locale].xml"],
  },
}
```

## ファイルパターン

ファイルパターンの[locale]プレースホルダーは、翻訳中に適切な言語コードに自動的に置き換えられます。例えば：

- locales/[locale].jsonは次のようになります：
  - locales/en.json
  - locales/es.json
  - locales/fr.json

各ファイルタイプに対して複数のパターンを含め、グロブパターンを使用できます：
```typescript
files: {
  json: {
    include: [
      "src/locales/[locale].json",
      "src/components/[locale]/*.json"
    ],
  },
}
```

---

## プロジェクトID（オプション）

Languineのクラウドサービスを使用している場合、設定にプロジェクトIDを含めることができます：

```typescript
export default defineConfig({
  projectId: "prj_your_project_id",
  locale: {
    // ... ロケール設定
  },
  files: {
    // ... ファイル設定
  },
});
```

---

## 翻訳状態の追跡

Languineは、自動的にlanguine.lockファイルを作成し、翻訳の状態を追跡します。このファイルは：

- どのコンテンツが翻訳されたかを追跡します
- ソースコンテンツが変更されたときに更新が必要なものを特定するのに役立ちます
- バージョン管理システムにコミットする必要があります

ロックファイルを手動で編集する必要はありません - Languineは翻訳コマンドを実行するときに自動的に管理します。

## ベストプラクティス

1. **バージョン管理**: 常にlanguine.config.tsとlanguine.lockの両方をリポジトリにコミットしてください
2. **ファイル整理**: 翻訳ファイルを一貫したディレクトリ構造で整理してください
3. **言語コード**: 互換性を確保するために標準の言語コードを使用してください
4. **パターン**: 不要なファイルを翻訳しないように特定のインクルードパターンを使用してください
5. **プロジェクトID**: クラウドサービスを使用している場合は、プロジェクトIDを安全に保ち、公開しないでください
