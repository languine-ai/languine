# CI/CD

Languineは、アプリケーションの翻訳を支援するCI/CD統合を提供します。

---

## 仕組み

LanguineのCI/CDワークフローは、アプリケーションの翻訳更新を自動化するための構造化されたプロセスに従います：

### 1. アクションの初期化
- アクションがトリガーされるとワークフローが開始されます
- 必要なGit設定と認証情報を設定します

### 2. Languineエンジン処理
- Languineエンジンがアプリケーションのコンテンツを処理します
- 設定された内容を使用して翻訳タスクを処理します

### 3. 変更検出
システムは翻訳の変更をチェックし、2つのパスのいずれかに従います：

**変更が見つかった場合：**
- 変更がリポジトリにコミットされます
- 変更が指定されたブランチにプッシュされます
- プルリクエスト（PR）が作成されます

**変更が見つからなかった場合：**
- ワークフローはPRを作成せずに完了します

### 4. プルリクエストの作成
変更が検出された場合、以下の内容でプルリクエストが作成されます：

**PR内容：**
- 翻訳更新を示す明確なタイトル
- 変更の詳細な説明
- 品質チェックの確認
- レビューのための指示
- ドキュメントとサポートへのリンク

**PR説明には以下が含まれます：**
- 翻訳更新の要約
- 変更された内容のリスト（更新された翻訳、用語の一貫性）
- レビューアへの次のステップ
- ドキュメントとサポートリソースへのリンク

ワークフローは自動化されるように設計されていますが、変更がコードベースにマージされる前に人間によるレビューと品質管理を可能にします。バージョン管理のベストプラクティスを維持し、翻訳がGitの履歴を通じて適切に追跡され、文書化されることを保証します。
