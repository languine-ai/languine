# iOS

Languineは、iOSアプリケーションの標準ローカリゼーション形式であるiOS Localizable.stringsファイルをサポートしています。この形式は、基本的なキーと値の文字列ペアを処理し、翻訳者にコンテキストを提供するためのコメントをサポートします。

---

## セットアップ

まず、プロジェクトのルートにlanguine.config.jsonまたはlanguine.config.tsの設定ファイルがあることを確認してください。以下はその例です：

```json
{
  "locale": {
    "source": "en",
    "targets": ["sv", "de", "fr"]
  },
  "files": {
    "ios": {
      "include": ["[locale].lproj/Localizable.strings"]
    }
  }
}
```

## 翻訳

設定が完了したら、次のコマンドを実行します：

```bash
npx languine@latest translate
```

このコマンドを実行すると、Languineは以下を行います：

- ソースのLocalizable.stringsファイル（例：en.lproj/Localizable.strings）を読み込みます
- 新しいまたは変更された翻訳文字列を検出します
- 対象言語の翻訳を生成します
- 対象言語ファイル（例：sv.lproj/Localizable.strings）を作成または更新します
- 既存のコメントとフォーマットを保持します
