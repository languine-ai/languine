# GitHub Actions

Dengan tindakan GitHub, Anda dapat dengan mudah menerjemahkan kode Anda hanya dengan mendorong kode Anda ke repositori, tindakan GitHub kemudian akan menerjemahkan kunci baru Anda, baik membuat permintaan tarik atau mengkomit perubahan Anda.
Ini menghemat banyak waktu dan usaha Anda sambil mempertahankan kualitas terjemahan yang tinggi.

---

## Instalasi

Untuk menggunakan Tindakan GitHub Languine di repositori Anda, Anda perlu:

1. Mengatur rahasia repositori Anda dengan kredensial Languine Anda
2. Membuat file alur kerja di repositori Anda

### Mengatur Rahasia Repositori

Tambahkan rahasia berikut di pengaturan repositori Anda (Pengaturan > Rahasia dan variabel > Tindakan):

- `LANGUINE_API_KEY`: Kunci API Languine Anda
- `LANGUINE_PROJECT_ID`: ID proyek Languine Anda

Anda dapat menemukan nilai-nilai ini di dasbor Languine Anda di bawah Pengaturan Proyek.



### Variabel Lingkungan

Tindakan GitHub menggunakan variabel lingkungan berikut:
- `LANGUINE_API_KEY` (Diperlukan): Kunci API Languine Anda untuk otentikasi
- `LANGUINE_PROJECT_ID` (Diperlukan): ID proyek Languine Anda
- `GITHUB_TOKEN` (Diperlukan): Secara otomatis disediakan oleh GitHub, digunakan untuk membuat PR

### Contoh Alur Kerja

Buat file baru .github/workflows/languine.yml di repositori Anda dengan konten berikut:

```ts title=".github/workflows/release.yml"
nama: Deploy Website

on:
  push:
    branches:
      - '**'
    paths:
      - apps/web/**

jobs:
  translate:
    runs-on: ubuntu-latest
    permissions:
      contents: write
      pull-requests: write
    steps:
      - uses: actions/checkout@v4
      
      - name: Tindakan Languine
        uses: languine/languine-action@v1
        env:
           GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
        with:
          api-key: ${{ secrets.LANGUINE_API_KEY }}
          project-id: ${{ secrets.LANGUINE_PROJECT_ID }}
```
 
 Alur kerja ini akan:
- Berjalan pada dorongan ke cabang utama
- Berjalan pada permintaan tarik ke cabang utama
- Dapat dipicu secara manual dari tab Tindakan
- Memeriksa kode Anda dan menjalankan proses terjemahan Languine
- Membuat permintaan tarik dengan terjemahan baru (dapat dikonfigurasi)

#### Opsi Permintaan Tarik

Opsi `create-pull-request` mengontrol apakah tindakan membuat permintaan tarik dengan perubahan atau mengkomitnya langsung ke cabang:

```typescript title=".github/workflows/languine.yml"
dengan:
  create-pull-request: true # buat permintaan tarik dengan perubahan
```

Ketika diatur ke `true`, tindakan akan:
- Membuat cabang baru dengan perubahan
- Membuat permintaan tarik untuk ditinjau
- Menambahkan label dan deskripsi yang sesuai

Ketika diatur ke `false` (default), tindakan akan:
- Mengkomit perubahan langsung ke cabang saat ini
- Mendorong perubahan tanpa membuat permintaan tarik
