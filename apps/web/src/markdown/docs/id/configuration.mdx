# Konfigurasi

Languine mendukung baik file konfigurasi TypeScript maupun JSON untuk mengelola pengaturan terjemahan Anda. Anda dapat memilih antara languine.config.ts untuk TypeScript atau languine.json untuk format JSON. Kedua file ini mendefinisikan bahasa sumber dan target Anda, serta file yang perlu diterjemahkan.

---

## Konfigurasi Dasar

Berikut adalah contoh dasar dalam kedua format:

```typescript title="languine.config.ts"
import { defineConfig } from "languine";

export default defineConfig({
  locale: {
    source: "en",
    targets: ["es", "fr", "de"],
  },
  files: {
    json: {
      include: ["locales/[locale].json"],
    },
  },
});
```

```json title="languine.json"
{
  "locale": {
    "source": "en",
    "targets": ["es", "fr", "de"]
  },
  "files": {
    "json": {
      "include": ["locales/[locale].json"]
    }
  }
}
```

Kedua format mendukung opsi konfigurasi yang sama. Pilih TypeScript untuk pemeriksaan tipe yang lebih baik dan dukungan IDE, atau JSON untuk kesederhanaan dan kompatibilitas alat yang lebih luas.

Mari kita uraikan bagian-bagian berbeda dari file-file ini.

---

## Konfigurasi Locale

Bagian locale mendefinisikan bahasa terjemahan Anda:

- source: Bahasa utama Anda untuk pembuatan konten (misalnya, "en" untuk Bahasa Inggris)
- targets: Sebuah array bahasa yang ingin Anda terjemahkan (misalnya, ["es", "fr", "de"] untuk Spanyol, Prancis, dan Jerman)

Languine mendukung berbagai kode bahasa, termasuk:
- Kode sederhana: en, es, fr, de
- Kode spesifik wilayah: pt-BR, zh-CN, zh-TW

## Konfigurasi File

Bagian files mendefinisikan file mana yang harus diterjemahkan oleh Languine. Anda dapat mengonfigurasi beberapa jenis dan pola file:

---

### File JSON
```typescript
files: {
  json: {
    include: ["locales/[locale].json"],
  },
}
```

---

### File Markdown/MDX
```typescript
files: {
  md: {
    include: ["docs/[locale]/*.md"],
  },
  mdx: {
    include: ["blog/[locale]/*.mdx"],
  },
}
```

---

### File Platform Seluler

Untuk iOS (Xcode):
```typescript
files: {
  "xcode-xcstrings": {
    include: ["Example/Localizable.xcstrings"],
  },
  // Atau untuk file .strings tradisional
  "xcode-strings": {
    include: ["Example/[locale].lproj/Localizable.strings"],
  },
}
```

---

Untuk Android:
```typescript
files: {
  android: {
    include: ["locales/[locale].xml"],
  },
}
```

## Pola File

Placeholder [locale] dalam pola file secara otomatis diganti dengan kode bahasa yang sesuai selama terjemahan. Misalnya:

- locales/[locale].json menjadi:
  - locales/en.json
  - locales/es.json
  - locales/fr.json

Anda dapat menyertakan beberapa pola untuk setiap jenis file dan menggunakan pola glob:
```typescript
files: {
  json: {
    include: [
      "src/locales/[locale].json",
      "src/components/[locale]/*.json"
    ],
  },
}
```

## ID Proyek (Opsional)

Jika Anda menggunakan layanan cloud Languine, Anda dapat menyertakan ID proyek Anda dalam konfigurasi:

```typescript
export default defineConfig({
  projectId: "prj_your_project_id",
  locale: {
    // ... konfigurasi locale
  },
  files: {
    // ... konfigurasi file
  },
});
```

## Pelacakan Status Terjemahan

Languine secara otomatis membuat dan memelihara file languine.lock untuk melacak status terjemahan Anda. File ini:

- Melacak konten mana yang telah diterjemahkan
- Membantu mengidentifikasi apa yang perlu diperbarui ketika konten sumber berubah
- Harus dikomit ke sistem kontrol versi Anda

Anda tidak perlu mengedit file kunci secara manual - Languine mengelolanya secara otomatis saat Anda menjalankan perintah terjemahan.

## Praktik Terbaik

1. **Kontrol Versi**: Selalu komit baik languine.config.ts maupun languine.lock ke repositori Anda
2. **Organisasi File**: Jaga agar file terjemahan Anda terorganisir dalam struktur direktori yang konsisten
3. **Kode Bahasa**: Gunakan kode bahasa standar untuk memastikan kompatibilitas
4. **Pola**: Gunakan pola penyertaan yang spesifik untuk menghindari menerjemahkan file yang tidak perlu
5. **ID Proyek**: Jaga agar ID proyek Anda aman dan jangan bagikan secara publik jika Anda menggunakan layanan cloud
