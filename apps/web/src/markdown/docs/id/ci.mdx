# CI/CD

Languine menyediakan integrasi CI/CD untuk membantu Anda menerjemahkan aplikasi Anda.

---

## Cara Kerjanya

Alur kerja CI/CD di Languine mengikuti proses terstruktur untuk mengotomatiskan pembaruan terjemahan dalam aplikasi Anda:

### 1. Inisialisasi Aksi
- Alur kerja dimulai ketika aksi dipicu
- Menyiapkan konfigurasi Git dan kredensial yang diperlukan

### 2. Pemrosesan Mesin Languine
- Mesin Languine memproses konten aplikasi Anda
- Menangani tugas terjemahan menggunakan pengaturan yang dikonfigurasi

### 3. Deteksi Perubahan
Sistem memeriksa adanya perubahan terjemahan dan mengikuti salah satu dari dua jalur:

**Ketika Perubahan Ditemukan:**
- Perubahan dikomit ke repositori
- Perubahan didorong ke cabang yang ditentukan
- Permintaan Tarik (PR) dibuat

**Ketika Tidak Ada Perubahan Ditemukan:**
- Alur kerja selesai tanpa membuat PR

### 4. Pembuatan Permintaan Tarik
Ketika perubahan terdeteksi, Permintaan Tarik dibuat dengan:

**Konten PR:**
- Judul yang jelas menunjukkan pembaruan terjemahan
- Deskripsi rinci tentang perubahan
- Konfirmasi pemeriksaan kualitas
- Instruksi untuk tinjauan
- Tautan ke dokumentasi dan dukungan

**Deskripsi PR Termasuk:**
- Ringkasan pembaruan terjemahan
- Daftar apa yang berubah (terjemahan yang diperbarui, konsistensi terminologi)
- Langkah selanjutnya untuk peninjau
- Tautan ke dokumentasi dan sumber daya dukungan
