# Cấu hình

Languine hỗ trợ cả tệp cấu hình TypeScript và JSON để quản lý thiết lập dịch thuật của bạn. Bạn có thể chọn languine.config.ts cho TypeScript hoặc languine.json cho định dạng JSON. Cả hai tệp đều xác định ngôn ngữ nguồn và ngôn ngữ đích của bạn, cũng như các tệp cần được dịch.

---

## Cấu hình cơ bản

Dưới đây là một ví dụ cơ bản ở cả hai định dạng:

```typescript title="languine.config.ts"
import { defineConfig } from "languine";

export default defineConfig({
  locale: {
    source: "en",
    targets: ["es", "fr", "de"],
  },
  files: {
    json: {
      include: ["locales/[locale].json"],
    },
  },
});
```

```json title="languine.json"
{
  "locale": {
    "source": "en",
    "targets": ["es", "fr", "de"]
  },
  "files": {
    "json": {
      "include": ["locales/[locale].json"]
    }
  }
}
```

Cả hai định dạng đều hỗ trợ các tùy chọn cấu hình giống nhau. Chọn TypeScript để có kiểm tra kiểu tốt hơn và hỗ trợ IDE, hoặc JSON cho sự đơn giản và khả năng tương thích công cụ rộng hơn.

Hãy cùng phân tích các phần khác nhau của những tệp này.

---

## Cấu hình ngôn ngữ

Phần ngôn ngữ xác định các ngôn ngữ dịch thuật của bạn:

- source: Ngôn ngữ chính của bạn để tạo nội dung (ví dụ: "en" cho tiếng Anh)
- targets: Một mảng các ngôn ngữ bạn muốn dịch sang (ví dụ: ["es", "fr", "de"] cho tiếng Tây Ban Nha, tiếng Pháp và tiếng Đức)

Languine hỗ trợ một loạt các mã ngôn ngữ, bao gồm:
- Mã đơn giản: en, es, fr, de
- Mã theo vùng: pt-BR, zh-CN, zh-TW

## Cấu hình tệp

Phần tệp xác định các tệp nào Languine nên dịch. Bạn có thể cấu hình nhiều loại tệp và mẫu:

---

### Tệp JSON
```typescript
files: {
  json: {
    include: ["locales/[locale].json"],
  },
}
```

---

### Tệp Markdown/MDX
```typescript
files: {
  md: {
    include: ["docs/[locale]/*.md"],
  },
  mdx: {
    include: ["blog/[locale]/*.mdx"],
  },
}
```

---

### Tệp nền tảng di động

Đối với iOS (Xcode):
```typescript
files: {
  "xcode-xcstrings": {
    include: ["Example/Localizable.xcstrings"],
  },
  // Hoặc cho các tệp .strings truyền thống
  "xcode-strings": {
    include: ["Example/[locale].lproj/Localizable.strings"],
  },
}
```

---

Đối với Android:
```typescript
files: {
  android: {
    include: ["locales/[locale].xml"],
  },
}
```

## Mẫu tệp

Biến [locale] trong các mẫu tệp sẽ tự động được thay thế bằng mã ngôn ngữ thích hợp trong quá trình dịch. Ví dụ:

- locales/[locale].json trở thành:
  - locales/en.json
  - locales/es.json
  - locales/fr.json

Bạn có thể bao gồm nhiều mẫu cho mỗi loại tệp và sử dụng mẫu glob:
```typescript
files: {
  json: {
    include: [
      "src/locales/[locale].json",
      "src/components/[locale]/*.json"
    ],
  },
}
```

## ID dự án (Tùy chọn)

Nếu bạn đang sử dụng dịch vụ đám mây của Languine, bạn có thể bao gồm ID dự án của mình trong cấu hình:

```typescript
export default defineConfig({
  projectId: "prj_your_project_id",
  locale: {
    // ... cấu hình ngôn ngữ
  },
  files: {
    // ... cấu hình tệp
  },
});
```

## Theo dõi trạng thái dịch thuật

Languine tự động tạo và duy trì tệp languine.lock để theo dõi trạng thái của các bản dịch của bạn. Tệp này:

- Theo dõi nội dung nào đã được dịch
- Giúp xác định những gì cần được cập nhật khi nội dung nguồn thay đổi
- Nên được cam kết vào hệ thống kiểm soát phiên bản của bạn

Bạn không cần phải chỉnh sửa thủ công tệp khóa - Languine quản lý nó tự động khi bạn chạy các lệnh dịch thuật.

## Thực hành tốt nhất

1. **Kiểm soát phiên bản**: Luôn cam kết cả languine.config.ts và languine.lock vào kho lưu trữ của bạn
2. **Tổ chức tệp**: Giữ cho các tệp dịch thuật của bạn được tổ chức trong một cấu trúc thư mục nhất quán
3. **Mã ngôn ngữ**: Sử dụng mã ngôn ngữ tiêu chuẩn để đảm bảo tính tương thích
4. **Mẫu**: Sử dụng các mẫu bao gồm cụ thể để tránh dịch các tệp không cần thiết
5. **ID dự án**: Giữ ID dự án của bạn an toàn và không chia sẻ công khai nếu bạn đang sử dụng dịch vụ đám mây
