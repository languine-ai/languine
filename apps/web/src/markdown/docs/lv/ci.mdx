# CI/CD

Languine nodrošina CI/CD integrāciju, lai palīdzētu jums tulkot jūsu lietojumprogrammu.

---

## Kā tas darbojas

CI/CD darba plūsma Languine seko strukturētam procesam, lai automatizētu tulkojumu atjauninājumus jūsu lietojumprogrammā:

### 1. Darbības inicializācija
- Darba plūsma sākas, kad tiek aktivizēta darbība
- Iestata nepieciešamo Git konfigurāciju un akreditācijas datus

### 2. Languine dzinēja apstrāde
- Languine dzinējs apstrādā jūsu lietojumprogrammas saturu
- Veic tulkojumu uzdevumus, izmantojot konfigurētās iestatījumus

### 3. Izmaiņu noteikšana
Sistēma pārbauda, vai ir kādas tulkojumu izmaiņas, un seko vienai no divām ceļiem:

**Kad tiek atrastas izmaiņas:**
- Izmaiņas tiek apstiprinātas repozitorijā
- Izmaiņas tiek nosūtītas uz norādīto zaru
- Tiek izveidots Pull Request (PR)

**Kad nav atrastas izmaiņas:**
- Darba plūsma pabeidzas, neveidojot PR

### 4. Pull Request izveide
Kad tiek noteiktas izmaiņas, tiek izveidots Pull Request ar:

**PR saturs:**
- Skaidrs nosaukums, kas norāda uz tulkojumu atjauninājumiem
- Detalizēts izmaiņu apraksts
- Kvalitātes pārbaudes apstiprinājumi
- Norādījumi pārskatīšanai
- Saites uz dokumentāciju un atbalstu

**PR apraksts ietver:**
- Tulkojumu atjauninājumu kopsavilkums
- Izmaiņu saraksts (atjauninātie tulkojumi, terminoloģijas konsekvence)
- Nākamie soļi pārskatītājiem
- Saites uz dokumentāciju un atbalsta resursiem
