# גיטאב אקשנס

עם האקשן של גיטאב, אתה יכול בקלות לתרגם את הקוד שלך פשוט על ידי דחיפת הקוד שלך למאגר, האקשן של גיטאב יתורגם אז את המפתחות החדשים שלך, או שיצור בקשת משיכה או יתחייב את השינויים שלך.
זה חוסך לך הרבה זמן ומאמץ תוך שמירה על תרגומים באיכות גבוהה.

---

## התקנה

כדי להשתמש באקשן של לנגווין במאגר שלך, תצטרך:

1. להגדיר את הסודות של המאגר שלך עם האישורים של לנגווין
2. ליצור קובץ זרימת עבודה במאגר שלך

### הגדרת סודות המאגר

הוסף את הסודות הבאים בהגדרות המאגר שלך (הגדרות > סודות ומשתנים > אקשנים):

- `LANGUINE_API_KEY`: מפתח ה-API שלך של לנגווין
- `LANGUINE_PROJECT_ID`: מזהה הפרויקט שלך של לנגווין

אתה יכול למצוא את הערכים הללו בלוח הבקרה של לנגווין תחת הגדרות הפרויקט.



### משתני סביבה

האקשן של גיטאב משתמש במשתני הסביבה הבאים:
- `LANGUINE_API_KEY` (נדרש): מפתח ה-API שלך של לנגווין לאימות
- `LANGUINE_PROJECT_ID` (נדרש): מזהה הפרויקט שלך של לנגווין 
- `GITHUB_TOKEN` (נדרש): מסופק אוטומטית על ידי גיטאב, משמש ליצירת בקשות משיכה

### דוגמת זרימת עבודה

צור קובץ חדש .github/workflows/languine.yml במאגר שלך עם התוכן הבא:

```ts title=".github/workflows/release.yml"
name: פריסת אתר

on:
  push:
    branches:
      - '**'
    paths:
      - apps/web/**

jobs:
  translate:
    runs-on: ubuntu-latest
    permissions:
      contents: write
      pull-requests: write
    steps:
      - uses: actions/checkout@v4
      
      - name: אקשן של לנגווין
        uses: languine/languine-action@v1
        env:
           GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
        with:
          api-key: ${{ secrets.LANGUINE_API_KEY }}
          project-id: ${{ secrets.LANGUINE_PROJECT_ID }}
```
 
 זרימת עבודה זו ת:
- תרוץ על דחיפות לענף הראשי
- תרוץ על בקשות משיכה לענף הראשי
- ניתן להפעיל ידנית מהטאב של האקשנים
- תבדוק את הקוד שלך ותפעיל את תהליך התרגום של לנגווין
- תיצור בקשת משיכה עם כל התרגומים החדשים (ניתן להגדיר)

#### אפשרות ספריית עבודה

אפשרות `working-directory` מאפשרת לך לציין היכן ממוקמים קבצי התרגום שלך. זה שימושי במיוחד במונורפוז שבהם התרגומים שלך עשויים להיות בחבילה או בתיקיית אפליקציה ספציפית. לדוגמה:

#### אפשרות בקשת משיכה

אפשרות `create-pull-request` שולטת האם האקשן ייצור בקשת משיכה עם השינויים או יתחייב אותם ישירות לענף:

```typescript title=".github/workflows/languine.yml"
with:
  create-pull-request: true # צור בקשת משיכה עם השינויים
```

כאשר מוגדר ל-`true`, האקשן:
- ייצור ענף חדש עם השינויים
- ייצור בקשת משיכה לסקירה
- יוסיף תוויות ותיאור מתאימים

כאשר מוגדר ל-`false` (ברירת מחדל), האקשן:
- יתחייב את השינויים ישירות לענף הנוכחי
- ידחוף את השינויים מבלי ליצור בקשת משיכה
