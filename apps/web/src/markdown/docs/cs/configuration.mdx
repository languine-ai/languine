# Konfigurace

Languine podporuje jak TypeScript, tak JSON konfigurační soubory pro správu vašeho překladového nastavení. Můžete si vybrat buď languine.config.ts pro TypeScript, nebo languine.json pro JSON formát. Oba soubory definují vaše zdrojové a cílové jazyky, stejně jako soubory, které je třeba přeložit.

---

## Základní konfigurace

Zde je základní příklad v obou formátech:

```typescript title="languine.config.ts"
import { defineConfig } from "languine";

export default defineConfig({
  locale: {
    source: "en",
    targets: ["es", "fr", "de"],
  },
  files: {
    json: {
      include: ["locales/[locale].json"],
    },
  },
});
```

```json title="languine.json"
{
  "locale": {
    "source": "en",
    "targets": ["es", "fr", "de"]
  },
  "files": {
    "json": {
      "include": ["locales/[locale].json"]
    }
  }
}
```

Oba formáty podporují stejné konfigurační možnosti. Zvolte TypeScript pro lepší kontrolu typů a podporu IDE, nebo JSON pro jednoduchost a širší kompatibilitu s nástroji.

Pojďme rozebrat různé sekce těchto souborů.

---

## Konfigurace jazyka

Sekce jazyka definuje vaše překladové jazyky:

- source: Váš primární jazyk pro vytváření obsahu (např. "en" pro angličtinu)
- targets: Pole jazyků, do kterých chcete překládat (např. ["es", "fr", "de"] pro španělštinu, francouzštinu a němčinu)

Languine podporuje širokou škálu jazykových kódů, včetně:
- Jednoduché kódy: en, es, fr, de
- Regionálně specifické kódy: pt-BR, zh-CN, zh-TW

## Konfigurace souborů

Sekce souborů definuje, které soubory by měl Languine překládat. Můžete konfigurovat více typů souborů a vzorů:

---

### JSON soubory
```typescript
files: {
  json: {
    include: ["locales/[locale].json"],
  },
}
```

---

### Markdown/MDX soubory
```typescript
files: {
  md: {
    include: ["docs/[locale]/*.md"],
  },
  mdx: {
    include: ["blog/[locale]/*.mdx"],
  },
}
```

---

### Soubory pro mobilní platformy

Pro iOS (Xcode):
```typescript
files: {
  "xcode-xcstrings": {
    include: ["Example/Localizable.xcstrings"],
  },
  // Nebo pro tradiční .strings soubory
  "xcode-strings": {
    include: ["Example/[locale].lproj/Localizable.strings"],
  },
}
```

---

Pro Android:
```typescript
files: {
  android: {
    include: ["locales/[locale].xml"],
  },
}
```

## Vzory souborů

Zástupce [locale] ve vzorech souborů je automaticky nahrazen příslušným jazykovým kódem během překladu. Například:

- locales/[locale].json se stává:
  - locales/en.json
  - locales/es.json
  - locales/fr.json

Můžete zahrnout více vzorů pro každý typ souboru a použít globální vzory:
```typescript
files: {
  json: {
    include: [
      "src/locales/[locale].json",
      "src/components/[locale]/*.json"
    ],
  },
}
```

## ID projektu (volitelné)

Pokud používáte cloudové služby Languine, můžete zahrnout své ID projektu do konfigurace:

```typescript
export default defineConfig({
  projectId: "prj_your_project_id",
  locale: {
    // ... konfigurace jazyka
  },
  files: {
    // ... konfigurace souborů
  },
});
```

## Sledování stavu překladu

Languine automaticky vytváří a udržuje soubor languine.lock pro sledování stavu vašich překladů. Tento soubor:

- Sledování, který obsah byl přeložen
- Pomáhá identifikovat, co je třeba aktualizovat, když se změní zdrojový obsah
- Měl by být zapsán do vašeho systému správy verzí

Nemusíte ručně upravovat soubor zámku - Languine ho spravuje automaticky, když spustíte překladové příkazy.

## Nejlepší praktiky

1. **Správa verzí**: Vždy zapište jak languine.config.ts, tak languine.lock do svého repozitáře
2. **Organizace souborů**: Udržujte své překladové soubory organizované v konzistentní struktuře adresářů
3. **Jazykové kódy**: Používejte standardní jazykové kódy pro zajištění kompatibility
4. **Vzory**: Používejte specifické vzory zahrnutí, abyste se vyhnuli překladu zbytečných souborů
5. **ID projektu**: Udržujte své ID projektu v bezpečí a nesdílejte ho veřejně, pokud používáte cloudové služby
