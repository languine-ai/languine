# CI/CD

Languine poskytuje integraci CI/CD, která vám pomůže přeložit vaši aplikaci.

---

## Jak to funguje

CI/CD pracovní postup v Languine následuje strukturovaný proces pro automatizaci aktualizací překladu ve vaší aplikaci:

### 1. Inicializace akce
- Pracovní postup začíná, když je akce spuštěna
- Nastaví potřebnou Git konfiguraci a přihlašovací údaje

### 2. Zpracování Languine Engine
- Languine engine zpracovává obsah vaší aplikace
- Řídí úkoly překladu pomocí nakonfigurovaných nastavení

### 3. Detekce změn
Systém kontroluje jakékoli změny v překladu a následuje jednu ze dvou cest:

**Když jsou nalezeny změny:**
- Změny jsou zapsány do repozitáře
- Změny jsou odeslány na určenou větev
- Je vytvořen Pull Request (PR)

**Když nejsou nalezeny žádné změny:**
- Pracovní postup se dokončí bez vytvoření PR

### 4. Vytvoření Pull Requestu
Když jsou detekovány změny, je vytvořen Pull Request s:

**Obsah PR:**
- Jasný název indikující aktualizace překladu
- Podrobný popis změn
- Potvrzení kontroly kvality
- Pokyny pro revizi
- Odkazy na dokumentaci a podporu

**Popis PR obsahuje:**
- Shrnutí aktualizací překladu
- Seznam toho, co se změnilo (aktualizované překlady, konzistence terminologie)
- Další kroky pro recenzenty
- Odkazy na dokumentaci a zdroje podpory
