# CLI

Languine은 애플리케이션 번역을 돕기 위한 CLI를 제공합니다.

## 명령어 개요

### translate

`translate` 명령어는 애플리케이션을 번역하는 데 사용됩니다.

```bash title="Terminal"
# 애플리케이션 번역 시작
languine translate

# 옵션:
#   --source-locale    원본 로케일 지정 (기본값: en)
#   --target-locale    번역할 대상 로케일 지정
#   --config          languine.config.js 파일 경로
```

### init

```bash title="Terminal"
# 새로운 Languine 프로젝트 초기화
languine init

# 이 작업은:
# - languine.config.js 파일 생성
# - 프로젝트 구조 설정
# - 필요한 디렉토리 초기화
```

### status

```bash title="Terminal"
# 번역 상태 확인
languine status

# 표시 내용:
# - 현재 번역 진행 상황
# - 누락된 번역
# - 번역 통계
```

### validate

```bash title="Terminal"
# 번역 파일 검증
languine validate

# 확인 사항:
# - 파일 구조
# - 누락된 키
# - 형식 일관성
```

### export

```bash title="Terminal"
# 번역 내보내기
languine export

# 옵션:
#   --format          내보내기 형식 (json, yaml 등)
#   --output         출력 디렉토리
```

### import

```bash title="Terminal"
# 외부 파일에서 번역 가져오기
languine import

# 옵션:
#   --source         원본 파일 또는 디렉토리
#   --format         가져오기 형식 (json, yaml 등)
```

## 전역 옵션

모든 명령어는 다음의 전역 옵션을 지원합니다:

```bash title="Terminal"
# 모든 명령어에 사용할 수 있는 전역 옵션
languine [command] [options]

# 일반 옵션:
#   --help, -h       도움말 정보 표시
#   --version, -v    버전 정보 표시
#   --verbose        자세한 로깅 활성화
#   --config         구성 파일 경로 지정
```
