# Ρύθμιση

Η Languine υποστηρίζει τόσο αρχεία ρύθμισης TypeScript όσο και JSON για τη διαχείριση της ρύθμισης μετάφρασης σας. Μπορείτε να επιλέξετε είτε το languine.config.ts για TypeScript είτε το languine.json για μορφή JSON. Και τα δύο αρχεία ορίζουν τις πηγές και τους στόχους γλώσσας σας, καθώς και τα αρχεία που χρειάζονται μετάφραση.

---

## Βασική Ρύθμιση

Ακολουθεί ένα βασικό παράδειγμα και στις δύο μορφές:

```typescript title="languine.config.ts"
import { defineConfig } from "languine";

export default defineConfig({
  locale: {
    source: "en",
    targets: ["es", "fr", "de"],
  },
  files: {
    json: {
      include: ["locales/[locale].json"],
    },
  },
});
```

```json title="languine.json"
{
  "locale": {
    "source": "en",
    "targets": ["es", "fr", "de"]
  },
  "files": {
    "json": {
      "include": ["locales/[locale].json"]
    }
  }
}
```

Και οι δύο μορφές υποστηρίζουν τις ίδιες επιλογές ρύθμισης. Επιλέξτε TypeScript για καλύτερο έλεγχο τύπων και υποστήριξη IDE, ή JSON για απλότητα και ευρύτερη συμβατότητα εργαλείων.

Ας αναλύσουμε τις διάφορες ενότητες αυτών των αρχείων.

---

## Ρύθμιση Γλώσσας

Η ενότητα γλώσσας ορίζει τις γλώσσες μετάφρασης σας:

- source: Η κύρια γλώσσα σας για τη δημιουργία περιεχομένου (π.χ., "en" για Αγγλικά)
- targets: Ένας πίνακας γλωσσών στις οποίες θέλετε να μεταφράσετε (π.χ., ["es", "fr", "de"] για Ισπανικά, Γαλλικά και Γερμανικά)

Η Languine υποστηρίζει μια ευρεία γκάμα κωδικών γλώσσας, συμπεριλαμβανομένων:
- Απλοί κωδικοί: en, es, fr, de
- Κωδικοί συγκεκριμένων περιοχών: pt-BR, zh-CN, zh-TW

## Ρύθμιση Αρχείων

Η ενότητα αρχείων ορίζει ποια αρχεία πρέπει να μεταφράσει η Languine. Μπορείτε να ρυθμίσετε πολλαπλούς τύπους και μοτίβα αρχείων:

---

### Αρχεία JSON
```typescript
files: {
  json: {
    include: ["locales/[locale].json"],
  },
}
```

---

### Αρχεία Markdown/MDX
```typescript
files: {
  md: {
    include: ["docs/[locale]/*.md"],
  },
  mdx: {
    include: ["blog/[locale]/*.mdx"],
  },
}
```

---

### Αρχεία Κινητής Πλατφόρμας

Για iOS (Xcode):
```typescript
files: {
  "xcode-xcstrings": {
    include: ["Example/Localizable.xcstrings"],
  },
  // Ή για παραδοσιακά αρχεία .strings
  "xcode-strings": {
    include: ["Example/[locale].lproj/Localizable.strings"],
  },
}
```

---

Για Android:
```typescript
files: {
  android: {
    include: ["locales/[locale].xml"],
  },
}
```

## Μοτίβα Αρχείων

Ο χώρος κράτησης [locale] στα μοτίβα αρχείων αντικαθίσταται αυτόματα με τον κατάλληλο κωδικό γλώσσας κατά τη διάρκεια της μετάφρασης. Για παράδειγμα:

- locales/[locale].json γίνεται:
  - locales/en.json
  - locales/es.json
  - locales/fr.json

Μπορείτε να συμπεριλάβετε πολλαπλά μοτίβα για κάθε τύπο αρχείου και να χρησιμοποιήσετε μοτίβα glob:
```typescript
files: {
  json: {
    include: [
      "src/locales/[locale].json",
      "src/components/[locale]/*.json"
    ],
  },
}
```

## ID Έργου (Προαιρετικό)

Εάν χρησιμοποιείτε τις υπηρεσίες cloud της Languine, μπορείτε να συμπεριλάβετε το ID του έργου σας στη ρύθμιση:

```typescript
export default defineConfig({
  projectId: "prj_your_project_id",
  locale: {
    // ... ρύθμιση γλώσσας
  },
  files: {
    // ... ρύθμιση αρχείων
  },
});
```

## Παρακολούθηση Κατάστασης Μετάφρασης

Η Languine δημιουργεί και διατηρεί αυτόματα ένα αρχείο languine.lock για να παρακολουθεί την κατάσταση των μεταφράσεών σας. Αυτό το αρχείο:

- Παρακολουθεί ποιο περιεχόμενο έχει μεταφραστεί
- Βοηθά στην αναγνώριση τι χρειάζεται να ενημερωθεί όταν αλλάζει το περιεχόμενο πηγής
- Πρέπει να δεσμευτεί στο σύστημα ελέγχου έκδοσης σας

Δεν χρειάζεται να επεξεργαστείτε το αρχείο κλειδώματος χειροκίνητα - η Languine το διαχειρίζεται αυτόματα όταν εκτελείτε εντολές μετάφρασης.

## Καλές Πρακτικές

1. **Έλεγχος Έκδοσης**: Πάντα να δεσμεύετε τόσο το languine.config.ts όσο και το languine.lock στο αποθετήριο σας
2. **Οργάνωση Αρχείων**: Διατηρήστε τα αρχεία μετάφρασής σας οργανωμένα σε μια συνεπή δομή καταλόγων
3. **Κωδικοί Γλώσσας**: Χρησιμοποιήστε τυπικούς κωδικούς γλώσσας για να διασφαλίσετε τη συμβατότητα
4. **Μοτίβα**: Χρησιμοποιήστε συγκεκριμένα μοτίβα συμπερίληψης για να αποφύγετε τη μετάφραση περιττών αρχείων
5. **ID Έργου**: Διατηρήστε το ID του έργου σας ασφαλές και μην το μοιράζεστε δημόσια αν χρησιμοποιείτε υπηρεσίες cloud
