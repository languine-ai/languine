# கட்டமைப்பு

Languine உங்கள் மொழிபெயர்ப்பு அமைப்பை நிர்வகிக்க TypeScript மற்றும் JSON கட்டமைப்பு கோப்புகளை ஆதரிக்கிறது. நீங்கள் TypeScript க்கான languine.config.ts அல்லது JSON வடிவத்திற்கான languine.json ஐ தேர்வு செய்யலாம். இரு கோப்புகளும் உங்கள் மூல மற்றும் இலக்கு மொழிகளை, மேலும் மொழிபெயர்க்க வேண்டிய கோப்புகளை வரையறுக்கின்றன.

---

## அடிப்படை கட்டமைப்பு

இரு வடிவங்களில் அடிப்படையான எடுத்துக்காட்டை இங்கே காணலாம்:

```typescript title="languine.config.ts"
import { defineConfig } from "languine";

export default defineConfig({
  locale: {
    source: "en",
    targets: ["es", "fr", "de"],
  },
  files: {
    json: {
      include: ["locales/[locale].json"],
    },
  },
});
```

```json title="languine.json"
{
  "locale": {
    "source": "en",
    "targets": ["es", "fr", "de"]
  },
  "files": {
    "json": {
      "include": ["locales/[locale].json"]
    }
  }
}
```

இரு வடிவங்களும் ஒரே கட்டமைப்பு விருப்பங்களை ஆதரிக்கின்றன. சிறந்த வகை சரிபார்ப்பு மற்றும் IDE ஆதரிக்க TypeScript ஐ தேர்வு செய்யவும், அல்லது எளிமை மற்றும் பரந்த கருவி ஒத்திசைவு க்காக JSON ஐ தேர்வு செய்யவும்.

இந்த கோப்புகளின் வெவ்வேறு பிரிவுகளை நாங்கள் உடைக்கலாம்.

---

## மொழி கட்டமைப்பு

மொழி பிரிவு உங்கள் மொழிபெயர்ப்பு மொழிகளை வரையறுக்கிறது:

- source: உள்ளடக்கம் உருவாக்குவதற்கான உங்கள் முதன்மை மொழி (எடுத்துக்காட்டாக, "en" என்பது ஆங்கிலம்)
- targets: நீங்கள் மொழிபெயர்க்க விரும்பும் மொழிகளின் வரிசை (எடுத்துக்காட்டாக, ["es", "fr", "de"] என்பது ஸ்பானிஷ், பிரெஞ்சு மற்றும் ஜெர்மன்)

Languine பல்வேறு மொழி குறியீடுகளை ஆதரிக்கிறது, இதில்:
- எளிய குறியீடுகள்: en, es, fr, de
- பிராந்திய குறிப்பிட்ட குறியீடுகள்: pt-BR, zh-CN, zh-TW

## கோப்பு கட்டமைப்பு

Languine மொழிபெயர்க்க வேண்டிய கோப்புகளை வரையறுக்கும் கோப்புகள் பிரிவு. நீங்கள் பல்வேறு கோப்பு வகைகள் மற்றும் மாதிரிகளை கட்டமைக்கலாம்:

---

### JSON கோப்புகள்
```typescript
files: {
  json: {
    include: ["locales/[locale].json"],
  },
}
```

---

### Markdown/MDX கோப்புகள்
```typescript
files: {
  md: {
    include: ["docs/[locale]/*.md"],
  },
  mdx: {
    include: ["blog/[locale]/*.mdx"],
  },
}
```

---

### மொபைல் தளம் கோப்புகள்

iOS (Xcode) க்கான:
```typescript
files: {
  "xcode-xcstrings": {
    include: ["Example/Localizable.xcstrings"],
  },
  // பாரம்பரிய .strings கோப்புகளுக்காக அல்லது
  "xcode-strings": {
    include: ["Example/[locale].lproj/Localizable.strings"],
  },
}
```

---

Android க்கான:
```typescript
files: {
  android: {
    include: ["locales/[locale].xml"],
  },
}
```

---

## கோப்பு மாதிரிகள்

கோப்பு மாதிரிகளில் [locale] இடம் மாற்றம் மொழிபெயர்ப்பு போது சரியான மொழி குறியீடுடன் தானாகவே மாற்றப்படுகிறது. எடுத்துக்காட்டாக:

- locales/[locale].json ஆக மாறுகிறது:
  - locales/en.json
  - locales/es.json
  - locales/fr.json

ஒவ்வொரு கோப்பு வகைக்கும் பல மாதிரிகளை நீங்கள் சேர்க்கலாம் மற்றும் glob மாதிரிகளை பயன்படுத்தலாம்:
```typescript
files: {
  json: {
    include: [
      "src/locales/[locale].json",
      "src/components/[locale]/*.json"
    ],
  },
}
```

## திட்ட ID (விருப்பம்)

நீங்கள் Languine இன் மேக சேவைகளைப் பயன்படுத்தினால், உங்கள் திட்ட ID ஐ கட்டமைப்பில் சேர்க்கலாம்:

```typescript
export default defineConfig({
  projectId: "prj_your_project_id",
  locale: {
    // ... மொழி கட்டமைப்பு
  },
  files: {
    // ... கோப்பு கட்டமைப்பு
  },
});
```

## மொழிபெயர்ப்பு நிலை கண்காணிப்பு

Languine தானாகவே languine.lock கோப்பை உருவாக்கி பராமரிக்கிறது, இது உங்கள் மொழிபெயர்ப்புகளின் நிலையை கண்காணிக்கிறது. இந்த கோப்பு:

- எந்த உள்ளடக்கம் மொழிபெயர்க்கப்பட்டுள்ளது என்பதை கண்காணிக்கிறது
- மூல உள்ளடக்கம் மாறும் போது புதுப்பிக்க வேண்டியதை அடையாளம் காண உதவுகிறது
- உங்கள் பதிப்பு கட்டுப்பாட்டு அமைப்புக்கு கட்டுப்படுத்தப்பட வேண்டும்

நீங்கள் கைவினை lock கோப்பை திருத்த தேவையில்லை - Languine மொழிபெயர்ப்பு கட்டளைகளை இயக்கும் போது தானாகவே அதை நிர்வகிக்கிறது.

## சிறந்த நடைமுறைகள்

1. **பதிப்பு கட்டுப்பாடு**: எப்போதும் languine.config.ts மற்றும் languine.lock இரண்டையும் உங்கள் சேமிப்பகத்தில் கட்டுப்படுத்தவும்
2. **கோப்பு ஒழுங்கமைப்பு**: உங்கள் மொழிபெயர்ப்பு கோப்புகளை ஒரே மாதிரியான அடிப்படையில் ஒழுங்கமைக்கவும்
3. **மொழி குறியீடுகள்**: ஒத்திசைவை உறுதி செய்ய தரநிலையான மொழி குறியீடுகளைப் பயன்படுத்தவும்
4. **மாதிரிகள்**: தேவையற்ற கோப்புகளை மொழிபெயர்க்காமல் இருக்க குறிப்பிட்ட உள்ளீட்டு மாதிரிகளைப் பயன்படுத்தவும்
5. **திட்ட ID**: உங்கள் திட்ட ID ஐ பாதுகாப்பாக வைத்திருங்கள் மற்றும் மேக சேவைகளைப் பயன்படுத்தினால் பொதுவாக பகிர வேண்டாம்
