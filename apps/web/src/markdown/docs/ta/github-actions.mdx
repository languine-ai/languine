# GitHub செயல்கள்

GitHub செயலுடன், நீங்கள் உங்கள் குறியீட்டை எளிதாக மொழிபெயர்க்கலாம், உங்கள் குறியீட்டை களஞ்சியத்திற்கு தள்ளுவதன் மூலம், GitHub செயலானது உங்கள் புதிய விசைகளை மொழிபெயர்க்கும், அல்லது ஒரு புல் கோரிக்கையை உருவாக்கும் அல்லது உங்கள் மாற்றங்களை உறுதிப்படுத்தும்.
இது உங்களுக்கு அதிக நேரம் மற்றும் முயற்சியைச் சேமிக்கிறது, மேலும் உயர் தரமான மொழிபெயர்ப்புகளை பராமரிக்கிறது.

---

## நிறுவல்

உங்கள் களஞ்சியத்தில் Languine GitHub செயலியைப் பயன்படுத்த, நீங்கள்:

1. உங்கள் Languine சான்றிதழ்களுடன் உங்கள் களஞ்சியத்தின் ரகசியங்களை அமைக்க வேண்டும்
2. உங்கள் களஞ்சியத்தில் ஒரு வேலைப்பாடு கோப்பை உருவாக்க வேண்டும்

### களஞ்சிய ரகசியங்களை அமைத்தல்

உங்கள் களஞ்சிய அமைப்புகளில் (அமைப்புகள் > ரகசியங்கள் மற்றும் மாறிலிகள் > செயல்கள்) கீழ்காணும் ரகசியங்களைச் சேர்க்கவும்:

- `LANGUINE_API_KEY`: உங்கள் Languine API விசை
- `LANGUINE_PROJECT_ID`: உங்கள் Languine திட்ட ID

இந்த மதிப்புகளை உங்கள் Languine டாஷ்போர்டில் திட்ட அமைப்புகளில் காணலாம்.



### சுற்றுப்புற மாறிலிகள்

GitHub செயலானது கீழ்காணும் சுற்றுப்புற மாறிலிகளைப் பயன்படுத்துகிறது:
- `LANGUINE_API_KEY` (தேவையானது): அங்கீகாரத்திற்கான உங்கள் Languine API விசை
- `LANGUINE_PROJECT_ID` (தேவையானது): உங்கள் Languine திட்ட ID 
- `GITHUB_TOKEN` (தேவையானது): GitHub மூலம் தானாக வழங்கப்படுகிறது, PR களை உருவாக்குவதற்குப் பயன்படுத்தப்படுகிறது

### எடுத்துக்காட்டு வேலைப்பாடு

உங்கள் களஞ்சியத்தில் .github/workflows/languine.yml என்ற புதிய கோப்பை கீழ்காணும் உள்ளடக்கத்துடன் உருவாக்கவும்:

```ts title=".github/workflows/release.yml"
name: வலைத்தளம் வெளியிடுதல்

on:
  push:
    branches:
      - '**'
    paths:
      - apps/web/**

jobs:
  translate:
    runs-on: ubuntu-latest
    permissions:
      contents: write
      pull-requests: write
    steps:
      - uses: actions/checkout@v4
      
      - name: Languine செயலி
        uses: languine/languine-action@v1
        env:
           GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
        with:
          api-key: ${{ secrets.LANGUINE_API_KEY }}
          project-id: ${{ secrets.LANGUINE_PROJECT_ID }}
```
 
 இந்த வேலைப்பாடு:
- முக்கிய கிளைக்கு தள்ளுதல்களில் இயங்கும்
- முக்கிய கிளைக்கு புல் கோரிக்கைகளில் இயங்கும்
- செயல்கள் தாவலில் கையால் தூண்டப்படலாம்
- உங்கள் குறியீட்டை சரிபார்க்கவும் மற்றும் Languine மொழிபெயர்ப்பு செயல்முறையை இயக்கவும்
- புதிய மொழிபெயர்ப்புகளை உள்ளடக்கிய புல் கோரிக்கையை உருவாக்கவும் (கட்டமைக்கக்கூடியது)

#### வேலை Directory விருப்பம்

`working-directory` விருப்பம் உங்கள் மொழிபெயர்ப்பு கோப்புகள் எங்கு உள்ளன என்பதை குறிப்பிட அனுமதிக்கிறது. இது குறிப்பிட்ட தொகுப்பு அல்லது பயன்பாட்டு அடைவில் உங்கள் மொழிபெயர்ப்புகள் இருக்கும்போது மிகவும் பயனுள்ளதாக இருக்கும். எடுத்துக்காட்டாக:

#### புல் கோரிக்கை விருப்பம்

`create-pull-request` விருப்பம் செயலி மாற்றங்களை உள்ளடக்கிய புல் கோரிக்கையை உருவாக்குகிறதா அல்லது நேரடியாக கிளைக்கு உறுதிப்படுத்துகிறதா என்பதை கட்டுப்படுத்துகிறது:

```typescript title=".github/workflows/languine.yml"
with:
  create-pull-request: true # மாற்றங்களுடன் ஒரு புல் கோரிக்கையை உருவாக்கவும்
```

`true` ஆக அமைக்கப்பட்டால், செயலி:
- மாற்றங்களுடன் புதிய கிளையை உருவாக்கும்
- மதிப்பீட்டிற்கான புல் கோரிக்கையை உருவாக்கும்
- பொருத்தமான லேபிள்கள் மற்றும் விளக்கங்களைச் சேர்க்கும்

`false` (இயல்பானது) ஆக அமைக்கப்பட்டால், செயலி:
- தற்போதைய கிளைக்கு நேரடியாக மாற்றங்களை உறுதிப்படுத்தும்
- புல் கோரிக்கையை உருவாக்காமல் மாற்றங்களை தள்ளும்
