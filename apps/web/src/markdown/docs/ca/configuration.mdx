# Configuració

Languine admet tant fitxers de configuració TypeScript com JSON per gestionar la teva configuració de traducció. Pots triar entre languine.config.ts per TypeScript o languine.json per al format JSON. Ambdós fitxers defineixen les teves llengües d'origen i de destí, així com els fitxers que necessiten ser traduïts.

---

## Configuració Bàsica

Aquí tens un exemple bàsic en ambdós formats:

```typescript title="languine.config.ts"
import { defineConfig } from "languine";

export default defineConfig({
  locale: {
    source: "en",
    targets: ["es", "fr", "de"],
  },
  files: {
    json: {
      include: ["locales/[locale].json"],
    },
  },
});
```

```json title="languine.json"
{
  "locale": {
    "source": "en",
    "targets": ["es", "fr", "de"]
  },
  "files": {
    "json": {
      "include": ["locales/[locale].json"]
    }
  }
}
```

Ambdós formats admeten les mateixes opcions de configuració. Tria TypeScript per a una millor comprovació de tipus i suport IDE, o JSON per a la simplicitat i una compatibilitat més àmplia amb eines.

Desglossem les diferents seccions d'aquests fitxers.

---

## Configuració de la Llengua

La secció de llengua defineix les teves llengües de traducció:

- source: La teva llengua principal per a la creació de contingut (per exemple, "en" per a l'anglès)
- targets: Un array de llengües a les quals vols traduir (per exemple, ["es", "fr", "de"] per a l'espanyol, el francès i l'alemany)

Languine admet una àmplia gamma de codis de llengua, incloent:
- Codis simples: en, es, fr, de
- Codis específics de regió: pt-BR, zh-CN, zh-TW

## Configuració de Fitxers

La secció de fitxers defineix quins fitxers ha de traduir Languine. Pots configurar múltiples tipus de fitxers i patrons:

---

### Fitxers JSON
```typescript
files: {
  json: {
    include: ["locales/[locale].json"],
  },
}
```

---

### Fitxers Markdown/MDX
```typescript
files: {
  md: {
    include: ["docs/[locale]/*.md"],
  },
  mdx: {
    include: ["blog/[locale]/*.mdx"],
  },
}
```

---

### Fitxers de Plataforma Mòbil

Per a iOS (Xcode):
```typescript
files: {
  "xcode-xcstrings": {
    include: ["Example/Localizable.xcstrings"],
  },
  // O per a fitxers .strings tradicionals
  "xcode-strings": {
    include: ["Example/[locale].lproj/Localizable.strings"],
  },
}
```

---

Per a Android:
```typescript
files: {
  android: {
    include: ["locales/[locale].xml"],
  },
}
```

## Patrons de Fitxers

El placeholder [locale] en els patrons de fitxer es reemplaça automàticament amb el codi de llengua apropiat durant la traducció. Per exemple:

- locales/[locale].json es converteix en:
  - locales/en.json
  - locales/es.json
  - locales/fr.json

Pots incloure múltiples patrons per a cada tipus de fitxer i utilitzar patrons glob:
```typescript
files: {
  json: {
    include: [
      "src/locales/[locale].json",
      "src/components/[locale]/*.json"
    ],
  },
}
```

## ID de Projecte (Opcional)

Si estàs utilitzant els serveis en núvol de Languine, pots incloure el teu ID de projecte a la configuració:

```typescript
export default defineConfig({
  projectId: "prj_your_project_id",
  locale: {
    // ... configuració de llengua
  },
  files: {
    // ... configuració de fitxers
  },
});
```

## Seguiment de l'Estat de Traducció

Languine crea i manté automàticament un fitxer languine.lock per fer un seguiment de l'estat de les teves traduccions. Aquest fitxer:

- Fa un seguiment de quin contingut ha estat traduït
- Ajuda a identificar què necessita ser actualitzat quan el contingut d'origen canvia
- Ha de ser compromès al teu sistema de control de versions

No cal que editis manualment el fitxer de bloqueig - Languine ho gestiona automàticament quan executes ordres de traducció.

## Millors Pràctiques

1. **Control de Versions**: Sempre compromet tant languine.config.ts com languine.lock al teu repositori
2. **Organització de Fitxers**: Mantingues els teus fitxers de traducció organitzats en una estructura de directoris consistent
3. **Codis de Llengua**: Utilitza codis de llengua estàndard per assegurar la compatibilitat
4. **Patrons**: Utilitza patrons d'inclusió específics per evitar traduir fitxers innecessaris
5. **ID de Projecte**: Mantingues el teu ID de projecte segur i no el comparteixis públicament si estàs utilitzant serveis en núvol
