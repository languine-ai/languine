# CLI

Languine bietet eine CLI, um Ihnen bei der Übersetzung Ihrer Anwendung zu helfen.

## Befehlsübersicht

### translate

Der Befehl `translate` wird verwendet, um Ihre Anwendung zu übersetzen.

```bash title="Terminal"
# Starten Sie die Übersetzung Ihrer Anwendung
languine translate

# Optionen:
#   --source-locale    Geben Sie die Quellsprache an (Standard: en)
#   --target-locale    Geben Sie die Zielsprache(n) an, in die übersetzt werden soll
#   --config          Pfad zu Ihrer languine.config.js-Datei
```

### init

```bash title="Terminal"
# Initialisieren Sie ein neues Languine-Projekt
languine init

# Dies wird:
# - Eine languine.config.js-Datei erstellen
# - Ihre Projektstruktur einrichten
# - Notwendige Verzeichnisse initialisieren
```

### status

```bash title="Terminal"
# Überprüfen Sie den Status Ihrer Übersetzungen
languine status

# Zeigt:
# - Aktuellen Übersetzungsfortschritt
# - Fehlende Übersetzungen
# - Übersetzungsstatistiken
```

### validate

```bash title="Terminal"
# Validieren Sie Ihre Übersetzungsdateien
languine validate

# Überprüft:
# - Dateistruktur
# - Fehlende Schlüssel
# - Formatkonsistenz
```

### export

```bash title="Terminal"
# Exportieren Sie Ihre Übersetzungen
languine export

# Optionen:
#   --format          Exportformat (json, yaml, usw.)
#   --output         Ausgabeverzeichnis
```

### import

```bash title="Terminal"
# Importieren Sie Übersetzungen aus externen Dateien
languine import

# Optionen:
#   --source         Quell-Datei oder Verzeichnis
#   --format         Importformat (json, yaml, usw.)
```

## Globale Optionen

Alle Befehle unterstützen die folgenden globalen Optionen:

```bash title="Terminal"
# Globale Optionen, die für alle Befehle verfügbar sind
languine [command] [options]

# Häufige Optionen:
#   --help, -h       Hilfeinformationen anzeigen
#   --version, -v    Versionsinformationen anzeigen
#   --verbose        Ausführliches Protokollieren aktivieren
#   --config         Pfad zur Konfigurationsdatei angeben
```
