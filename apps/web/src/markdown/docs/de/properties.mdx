# Eigenschaften

Languine unterstützt Java Properties-Dateien, das Standardlokalisierungsformat für Java-Anwendungen. Dieses Format verarbeitet Unicode-Escaping und unterstützt sowohl Schlüssel-Wert-Paare als auch Kommentare.

---

## Einrichtung

Stellen Sie zunächst sicher, dass Sie eine languine.config.json oder languine.config.ts Konfigurationsdatei im Stammverzeichnis Ihres Projekts haben. Hier ist ein Beispiel:

```json
{
  "locale": {
    "source": "en",
    "targets": ["sv", "de", "fr"]
  },
  "files": {
    "properties": {
      "include": ["src/main/resources/messages_[locale].properties"]
    }
  }
}
```

## Übersetzen

Mit Ihrer Konfiguration ausgeführt, führen Sie aus:

```bash
npx languine@latest translate
```

Wenn Sie diesen Befehl ausführen, wird Languine:

- Ihre Quell-Properties-Datei laden (z. B. messages_en.properties)
- Neue oder geänderte Übersetzungsstrings erkennen
- Übersetzungen für Ihre Zielsprache generieren
- Die Properties-Dateien für die Zielsprache erstellen oder aktualisieren (z. B. messages_sv.properties)
- Unicode-Escaping automatisch behandeln
- Kommentare und Formatierung beibehalten
