# Markdown

Languine unterstützt die Lokalisierung von Markdown-Dateien, ideal für Dokumentationen, Blogbeiträge und inhaltsreiche Websites. Dieses Format bewahrt die Markdown-Syntax während der Übersetzung des Inhalts, einschließlich der Unterstützung von Front Matter.

---

## Einrichtung

Stellen Sie zunächst sicher, dass Sie eine languine.config.json oder languine.config.ts Konfigurationsdatei im Stammverzeichnis Ihres Projekts haben. Hier ist ein Beispiel:

```json
{
  "locale": {
    "source": "en",
    "targets": ["sv", "de", "fr"]
  },
  "files": {
    "md": {
      "include": ["content/[locale]/**/*.md"],
      "frontMatter": true,  // Optional: Standardmäßig auf true
      "translateFrontMatterFields": ["title", "description"]  // Optional: geben Sie an, welche Front Matter-Felder übersetzt werden sollen
    }
  }
}
```

## Übersetzung

Mit Ihrer Konfiguration können Sie Folgendes ausführen:

```bash
npx languine@latest translate
```

Wenn Sie diesen Befehl ausführen, wird Languine:

- Ihre Quell-Markdown-Dateien laden (z. B. content/en/*.md)
- Alle neuen oder geänderten Inhalte erkennen
- Übersetzungen für Ihre Zielsprache generieren
- Die Dateien der Zielsprache erstellen oder aktualisieren (z. B. content/sv/*.md)
- Die Markdown-Formatierung und -Struktur beibehalten
- Die Übersetzungen des Front Matter verwalten
- Links und Verweise beibehalten
- Codeblöcke unverändert lassen
- HTML innerhalb von Markdown bewahren
