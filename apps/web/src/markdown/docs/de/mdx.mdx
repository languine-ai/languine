# Markdown (MDX)

Languine unterstützt die Lokalisierung von Markdown (MDX)-Dateien, ideal für Dokumentationen, Blogbeiträge und inhaltsreiche Websites. Dieses Format bewahrt die Markdown-Syntax während der Übersetzung des Inhalts, einschließlich der Unterstützung von Front Matter.

---

## Einrichtung

Zuerst stellen Sie sicher, dass Sie eine languine.config.json oder languine.config.ts Konfigurationsdatei im Stammverzeichnis Ihres Projekts haben. Hier ist ein Beispiel:

```json
{
  "locale": {
    "source": "en",
    "targets": ["sv", "de", "fr"]
  },
  "files": {
    "mdx": {
      "include": ["content/[locale]/**/*.mdx"]
    }
  }
}
```

## Übersetzung

Mit Ihrer Konfiguration ausgeführt, führen Sie aus:

```bash
npx languine@latest translate
```

Wenn Sie diesen Befehl ausführen, wird Languine:

- Ihre Quell-Markdown-Dateien laden (z. B. content/en/*.mdx)
- Alle neuen oder geänderten Inhalte erkennen
- Übersetzungen für Ihre Zielsprache generieren
- Die Zielsprachendateien erstellen oder aktualisieren (z. B. content/sv/*.mdx)
- Die Markdown-Formatierung und -struktur bewahren
- Die Übersetzungen des Front Matter verwalten
- Links und Referenzen beibehalten
- Codeblöcke unverändert lassen
- HTML innerhalb von Markdown bewahren
