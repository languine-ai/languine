# React Email

Languine obsługuje szablony e-mailowe React, umożliwiając zarządzanie tłumaczeniami dla szablonów e-mailowych stworzonych za pomocą React Email. Ta integracja obsługuje zarówno treść tekstową, jak i atrybuty HTML, zachowując strukturę komponentów React.

---

## Konfiguracja

Najpierw upewnij się, że masz plik konfiguracyjny languine.json w katalogu głównym projektu. Oto przykład:

```json
{
  "locale": {
    "source": "en",
    "targets": ["sv", "de", "fr"]
  },
  "files": {
    "react-email": {
      "include": ["emails/[locale]/*.tsx"]
    }
  }
}
```

## Tłumaczenie

Po ustawieniu konfiguracji uruchom:

```bash
npx languine@latest translate
```

Kiedy uruchomisz to polecenie, Languine:

- Załaduje twoje źródłowe szablony e-mailowe React (np. emails/en/*.tsx)
- Wykryje wszelkie nowe lub zmodyfikowane ciągi tłumaczeń
- Wygeneruje tłumaczenia dla twoich docelowych języków
- Utworzy lub zaktualizuje szablony e-mailowe w docelowym języku (np. emails/sv/*.tsx)
- Zachowa strukturę komponentów React i właściwości
- Utrzyma formatowanie i styl HTML
- Obsłuży interpolację i dynamiczną treść
