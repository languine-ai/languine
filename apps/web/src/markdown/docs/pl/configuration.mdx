# Konfiguracja

Languine obsługuje zarówno pliki konfiguracyjne TypeScript, jak i JSON, aby zarządzać ustawieniami tłumaczeń. Możesz wybrać albo languine.config.ts dla TypeScript, albo languine.json dla formatu JSON. Oba pliki definiują twoje źródłowe i docelowe języki, a także pliki, które muszą być przetłumaczone.

---

## Podstawowa konfiguracja

Oto podstawowy przykład w obu formatach:

```typescript title="languine.config.ts"
import { defineConfig } from "languine";

export default defineConfig({
  locale: {
    source: "en",
    targets: ["es", "fr", "de"],
  },
  files: {
    json: {
      include: ["locales/[locale].json"],
    },
  },
});
```

```json title="languine.json"
{
  "locale": {
    "source": "en",
    "targets": ["es", "fr", "de"]
  },
  "files": {
    "json": {
      "include": ["locales/[locale].json"]
    }
  }
}
```

Oba formaty obsługują te same opcje konfiguracyjne. Wybierz TypeScript dla lepszej kontroli typów i wsparcia IDE, lub JSON dla prostoty i szerszej kompatybilności narzędzi.

Rozłóżmy różne sekcje tych plików.

## Konfiguracja lokalizacji

Sekcja lokalizacji definiuje twoje języki tłumaczeń:

- source: Twój podstawowy język do tworzenia treści (np. "en" dla angielskiego)
- targets: Tablica języków, na które chcesz przetłumaczyć (np. ["es", "fr", "de"] dla hiszpańskiego, francuskiego i niemieckiego)

Languine obsługuje szeroki zakres kodów językowych, w tym:
- Proste kody: en, es, fr, de
- Kody specyficzne dla regionu: pt-BR, zh-CN, zh-TW

## Konfiguracja plików

Sekcja plików definiuje, które pliki Languine powinien przetłumaczyć. Możesz skonfigurować wiele typów plików i wzorców:

---

### Pliki JSON
```typescript
files: {
  json: {
    include: ["locales/[locale].json"],
  },
}
```

---

### Pliki Markdown/MDX
```typescript
files: {
  md: {
    include: ["docs/[locale]/*.md"],
  },
  mdx: {
    include: ["blog/[locale]/*.mdx"],
  },
}
```

---

### Pliki platform mobilnych

Dla iOS (Xcode):
```typescript
files: {
  "xcode-xcstrings": {
    include: ["Example/Localizable.xcstrings"],
  },
  // Lub dla tradycyjnych plików .strings
  "xcode-strings": {
    include: ["Example/[locale].lproj/Localizable.strings"],
  },
}
```

---

Dla Androida:
```typescript
files: {
  android: {
    include: ["locales/[locale].xml"],
  },
}
```

## Wzorce plików

Zmienna [locale] w wzorcach plików jest automatycznie zastępowana odpowiednim kodem językowym podczas tłumaczenia. Na przykład:

- locales/[locale].json staje się:
  - locales/en.json
  - locales/es.json
  - locales/fr.json

Możesz uwzględnić wiele wzorców dla każdego typu pliku i używać wzorców glob:
```typescript
files: {
  json: {
    include: [
      "src/locales/[locale].json",
      "src/components/[locale]/*.json"
    ],
  },
}
```

## ID projektu (opcjonalne)

Jeśli korzystasz z usług chmurowych Languine, możesz uwzględnić swoje ID projektu w konfiguracji:

```typescript
export default defineConfig({
  projectId: "prj_your_project_id",
  locale: {
    // ... konfiguracja lokalizacji
  },
  files: {
    // ... konfiguracja plików
  },
});
```

---

## Śledzenie stanu tłumaczeń

Languine automatycznie tworzy i utrzymuje plik languine.lock, aby śledzić stan twoich tłumaczeń. Ten plik:

- Śledzi, która treść została przetłumaczona
- Pomaga zidentyfikować, co należy zaktualizować, gdy zmienia się treść źródłowa
- Powinien być zatwierdzony w twoim systemie kontroli wersji

Nie musisz ręcznie edytować pliku blokady - Languine zarządza nim automatycznie, gdy uruchamiasz polecenia tłumaczenia.

## Najlepsze praktyki

1. **Kontrola wersji**: Zawsze zatwierdzaj zarówno languine.config.ts, jak i languine.lock w swoim repozytorium
2. **Organizacja plików**: Utrzymuj swoje pliki tłumaczeń w zorganizowanej, spójnej strukturze katalogów
3. **Kody językowe**: Używaj standardowych kodów językowych, aby zapewnić kompatybilność
4. **Wzorce**: Używaj specyficznych wzorców dołączania, aby uniknąć tłumaczenia niepotrzebnych plików
5. **ID projektu**: Zachowaj swoje ID projektu w bezpiecznym miejscu i nie udostępniaj go publicznie, jeśli korzystasz z usług chmurowych
