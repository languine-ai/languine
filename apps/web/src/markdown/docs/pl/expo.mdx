# Expo

Languine wspiera lokalizację Expo, integrując się płynnie z systemem i18n Expo. Ten format działa zarówno z projektami JavaScript, jak i TypeScript, wspierając zagnieżdżone struktury oraz pluralizację.

---

## Konfiguracja

Najpierw upewnij się, że masz plik konfiguracyjny languine.json w katalogu głównym projektu. Oto przykład:

```json
{
  "locale": {
    "source": "en",
    "targets": ["sv", "de", "fr"]
  },
  "files": {
    "expo": {
      "include": ["assets/locales/[locale].json"]
    }
  }
}
```

## Tłumaczenie

Gdy masz ustawioną konfigurację, uruchom:

```bash
npx languine@latest translate
```

Kiedy uruchomisz to polecenie, Languine:

- Załaduje twój plik źródłowy z tłumaczeniem (np. assets/locales/en.json)
- Wykryje wszelkie nowe lub zmodyfikowane ciągi tłumaczeń
- Wygeneruje tłumaczenia dla twoich docelowych języków
- Utworzy lub zaktualizuje pliki języków docelowych (np. assets/locales/sv.json)
- Zachowa zagnieżdżoną strukturę oraz zasady pluralizacji
- Utrzyma zgodność z expo-localization
