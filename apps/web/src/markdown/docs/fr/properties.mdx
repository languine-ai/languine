# Propriétés

Languine prend en charge les fichiers de propriétés Java, le format de localisation standard pour les applications Java. Ce format gère l'échappement Unicode et prend en charge à la fois les paires clé-valeur et les commentaires.

---

## Configuration

Tout d'abord, assurez-vous d'avoir un fichier de configuration languine.json à la racine de votre projet. Voici un exemple :

```json
{
  "locale": {
    "source": "en",
    "targets": ["sv", "de", "fr"]
  },
  "files": {
    "properties": {
      "include": ["src/main/resources/messages_[locale].properties"]
    }
  }
}
```

## Traduction

Avec votre configuration en place, exécutez :

```bash
npx languine@latest translate
```

Lorsque vous exécutez cette commande, Languine va :

- Charger votre fichier de propriétés source (par exemple, messages_en.properties)
- Détecter toute nouvelle chaîne de traduction ou toute chaîne modifiée
- Générer des traductions pour vos langues cibles
- Créer ou mettre à jour les fichiers de propriétés des langues cibles (par exemple, messages_sv.properties)
- Gérer automatiquement l'échappement Unicode
- Préserver les commentaires et le formatage
