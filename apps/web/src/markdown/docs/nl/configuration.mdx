# Configuratie

Languine ondersteunt zowel TypeScript- als JSON-configuratiebestanden om uw vertalingsinstellingen te beheren. U kunt kiezen voor languine.config.ts voor TypeScript of languine.config.json voor JSON-indeling. Beide bestanden definiÃ«ren uw bron- en doeltalen, evenals de bestanden die vertaald moeten worden.

---

## Basisconfiguratie

Hier is een basisvoorbeeld in beide formaten:

```typescript title="languine.config.ts"
import { defineConfig } from "languine";

export default defineConfig({
  locale: {
    source: "en",
    targets: ["es", "fr", "de"],
  },
  files: {
    json: {
      include: ["locales/[locale].json"],
    },
  },
});
```

```json title="languine.config.json"
{
  "locale": {
    "source": "en",
    "targets": ["es", "fr", "de"]
  },
  "files": {
    "json": {
      "include": ["locales/[locale].json"]
    }
  }
}
```

Beide formaten ondersteunen dezelfde configuratieopties. Kies TypeScript voor betere typecontrole en IDE-ondersteuning, of JSON voor eenvoud en bredere toolcompatibiliteit.

Laten we de verschillende secties van deze bestanden bekijken.

---

## Locale-configuratie

De locale-sectie definieert uw vertalingen:

- source: Uw primaire taal voor contentcreatie (bijv. "en" voor Engels)
- targets: Een array van talen waarin u wilt vertalen (bijv. ["es", "fr", "de"] voor Spaans, Frans en Duits)

Languine ondersteunt een breed scala aan taalcodes, waaronder:
- Eenvoudige codes: en, es, fr, de
- Regio-specifieke codes: pt-BR, zh-CN, zh-TW

## Bestandsconfiguratie

De bestanden-sectie definieert welke bestanden Languine moet vertalen. U kunt meerdere bestandstypen en patronen configureren:

---

### JSON-bestanden
```typescript
files: {
  json: {
    include: ["locales/[locale].json"],
  },
}
```

---

### Markdown/MDX-bestanden
```typescript
files: {
  md: {
    include: ["docs/[locale]/*.md"],
  },
  mdx: {
    include: ["blog/[locale]/*.mdx"],
  },
}
```

---

### Mobiele platformbestanden

Voor iOS (Xcode):
```typescript
files: {
  "xcode-xcstrings": {
    include: ["Example/Localizable.xcstrings"],
  },
  // Of voor traditionele .strings-bestanden
  "xcode-strings": {
    include: ["Example/[locale].lproj/Localizable.strings"],
  },
}
```

---

Voor Android:
```typescript
files: {
  android: {
    include: ["locales/[locale].xml"],
  },
}
```

## Bestands patronen

De [locale] plaatsvervanger in bestands patronen wordt automatisch vervangen door de juiste taalcodes tijdens de vertaling. Bijvoorbeeld:

- locales/[locale].json wordt:
  - locales/en.json
  - locales/es.json
  - locales/fr.json

U kunt meerdere patronen voor elk bestandstype opnemen en glob-patronen gebruiken:
```typescript
files: {
  json: {
    include: [
      "src/locales/[locale].json",
      "src/components/[locale]/*.json"
    ],
  },
}
```

## Project-ID (optioneel)

Als u gebruikmaakt van de cloudservices van Languine, kunt u uw project-ID in de configuratie opnemen:

```typescript
export default defineConfig({
  projectId: "prj_your_project_id",
  locale: {
    // ... locale-configuratie
  },
  files: {
    // ... bestanden-configuratie
  },
});
```

## Volgen van vertaalstatus

Languine maakt automatisch een languine.lock-bestand aan en onderhoudt dit om de status van uw vertalingen bij te houden. Dit bestand:

- Houdt bij welke inhoud is vertaald
- Helpt te identificeren wat moet worden bijgewerkt wanneer de broninhoud verandert
- Moet worden gecommit naar uw versiebeheersysteem

U hoeft het lock-bestand niet handmatig te bewerken - Languine beheert dit automatisch wanneer u vertaalopdrachten uitvoert.

## Beste praktijken

1. **Versiebeheer**: Commit altijd zowel languine.config.ts als languine.lock naar uw repository
2. **Bestandsorganisatie**: Houd uw vertaalbestanden georganiseerd in een consistente mappenstructuur
3. **Taalcodes**: Gebruik standaardtaalcodes om compatibiliteit te waarborgen
4. **Patronen**: Gebruik specifieke inclusiepatronen om te voorkomen dat onnodige bestanden worden vertaald
5. **Project-ID**: Houd uw project-ID veilig en deel deze niet openbaar als u gebruikmaakt van cloudservices, gebruik in plaats daarvan de omgevingsvariabele `LANGUINE_PROJECT_ID`.
