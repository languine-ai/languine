# CI/CD

Languine предоставляет интеграцию CI/CD, чтобы помочь вам перевести ваше приложение.

---

## Как это работает

Рабочий процесс CI/CD в Languine следует структурированному процессу для автоматизации обновлений перевода в вашем приложении:

### 1. Инициализация действия
- Рабочий процесс начинается, когда действие запускается
- Настраивает необходимую конфигурацию Git и учетные данные

### 2. Обработка движка Languine
- Движок Languine обрабатывает содержимое вашего приложения
- Выполняет задачи перевода с использованием настроенных параметров

### 3. Обнаружение изменений
Система проверяет наличие изменений в переводе и следует одному из двух путей:

**Когда изменения найдены:**
- Изменения фиксируются в репозитории
- Изменения отправляются в указанную ветку
- Создается Pull Request (PR)

**Когда изменения не найдены:**
- Рабочий процесс завершается без создания PR

### 4. Создание Pull Request
Когда изменения обнаружены, создается Pull Request с:

**Содержимое PR:**
- Ясный заголовок, указывающий на обновления перевода
- Подробное описание изменений
- Подтверждения проверки качества
- Инструкции для рецензирования
- Ссылки на документацию и поддержку

**Описание PR включает:**
- Резюме обновлений перевода
- Список изменений (обновленные переводы, согласованность терминологии)
- Следующие шаги для рецензентов
- Ссылки на документацию и ресурсы поддержки
