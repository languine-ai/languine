# GitHub Actions

С GitHub действието можете лесно да преведете кода си, просто като го качите в хранилището. GitHub действието след това ще преведе новите ви ключове, ще създаде pull request или ще запише промените ви.
Това ви спестява много време и усилия, като същевременно поддържа високо качество на преводите.

---

## Инсталация

За да използвате GitHub действието на Languine в хранилището си, трябва да:

1. Настроите тайните на хранилището си с вашите Languine идентификационни данни
2. Създадете файл за работен поток в хранилището си

### Настройка на тайните на хранилището

Добавете следните тайни в настройките на хранилището си (Настройки > Тайни и променливи > Действия):

- `LANGUINE_API_KEY`: Вашият Languine API ключ
- `LANGUINE_PROJECT_ID`: Вашият Languine проект ID

Можете да намерите тези стойности в таблото на Languine под Настройки на проекта.



### Променливи на средата

GitHub действието използва следните променливи на средата:
- `LANGUINE_API_KEY` (Задължително): Вашият Languine API ключ за удостоверяване
- `LANGUINE_PROJECT_ID` (Задължително): Вашият Languine проект ID 
- `GITHUB_TOKEN` (Задължително): Автоматично предоставен от GitHub, използван за създаване на PR

### Примерен работен поток

Създайте нов файл .github/workflows/languine.yml в хранилището си с следното съдържание:

```ts title=".github/workflows/release.yml"
name: Разгръщане на уебсайт

on:
  push:
    branches:
      - '**'
    paths:
      - apps/web/**

jobs:
  translate:
    runs-on: ubuntu-latest
    permissions:
      contents: write
      pull-requests: write
    steps:
      - uses: actions/checkout@v4
      
      - name: Languine действие
        uses: languine/languine-action@v1
        env:
           GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
        with:
          api-key: ${{ secrets.LANGUINE_API_KEY }}
          project-id: ${{ secrets.LANGUINE_PROJECT_ID }}
```
 
 Този работен поток ще:
- Работи при качвания в основния клон
- Работи при pull requests към основния клон
- Може да бъде задействан ръчно от таба Действия
- Изтегли кода ви и да стартира процеса на превод на Languine
- Създаде pull request с новите преводи (конфигурируемо)

#### Опция за Pull Request

Опцията `create-pull-request` контролира дали действието създава pull request с промените или ги записва директно в клона:

```typescript title=".github/workflows/languine.yml"
with:
  create-pull-request: true # създайте pull request с промените
```

Когато е зададена на `true`, действието ще:
- Създаде нов клон с промените
- Създаде pull request за преглед
- Добави подходящи етикети и описание

Когато е зададена на `false` (по подразбиране), действието ще:
- Запише промените директно в текущия клон
- Избута промените без да създава pull request
