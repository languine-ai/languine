# CLI

Languine предоставя CLI, за да ви помогне да преведете вашето приложение.

## Преглед на командите

### translate

Командата `translate` се използва за превод на вашето приложение.

```bash title="Terminal"
# Започнете превода на вашето приложение
languine translate

# Опции:
#   --source-locale    Укажете изходния език (по подразбиране: en)
#   --target-locale    Укажете целевия език(и) за превод
#   --config          Път до вашия languine.config.js файл
```

### init

```bash title="Terminal"
# Инициализирайте нов проект Languine
languine init

# Това ще:
# - Създаде файл languine.config.js
# - Настрои структурата на вашия проект
# - Инициализира необходимите директории
```

### status

```bash title="Terminal"
# Проверете статуса на вашите преводи
languine status

# Показва:
# - Текущ напредък на превода
# - Липсващи преводи
# - Статистика за превода
```

### validate

```bash title="Terminal"
# Валидирайте вашите файлове с преводи
languine validate

# Проверява:
# - Структура на файла
# - Липсващи ключове
# - Консистентност на формата
```

### export

```bash title="Terminal"
# Експортирайте вашите преводи
languine export

# Опции:
#   --format          Формат на експорта (json, yaml и др.)
#   --output         Изходна директория
```

### import

```bash title="Terminal"
# Импортирайте преводи от външни файлове
languine import

# Опции:
#   --source         Изходен файл или директория
#   --format         Формат на импорта (json, yaml и др.)
```

## Глобални опции

Всички команди поддържат следните глобални опции:

```bash title="Terminal"
# Глобални опции, налични за всички команди
languine [command] [options]

# Общи опции:
#   --help, -h       Показване на информация за помощ
#   --version, -v    Показване на информация за версията
#   --verbose        Активиране на подробен логинг
#   --config         Укажете пътя до конфигурационния файл
```
