# CI/CD

Languine предоставя интеграция за CI/CD, за да ви помогне да преведете приложението си.

---

## Как работи

CI/CD работният процес в Languine следва структурирана процедура за автоматизиране на актуализациите на преводите в приложението ви:

### 1. Инициализация на действието
- Работният процес започва, когато действието бъде задействано
- Настройва необходимата конфигурация на Git и удостоверения

### 2. Обработка от Languine Engine
- Languine engine обработва съдържанието на вашето приложение
- Изпълнява задачите по превод, използвайки конфигурираните настройки

### 3. Откриване на промени
Системата проверява за всякакви промени в превода и следва един от двата пътя:

**Когато бъдат намерени промени:**
- Промените се ангажират с хранилището
- Промените се изпращат към указаното клонче
- Създава се Pull Request (PR)

**Когато не бъдат намерени промени:**
- Работният процес завършва без да създаде PR

### 4. Създаване на Pull Request
Когато бъдат открити промени, се създава Pull Request с:

**Съдържание на PR:**
- Ясен заглавие, указващо актуализации на преводите
- Подробно описание на промените
- Потвърждения за проверка на качеството
- Инструкции за преглед
- Връзки към документация и поддръжка

**Описание на PR включва:**
- Резюме на актуализациите на преводите
- Списък на промените (актуализирани преводи, последователност на терминологията)
- Следващи стъпки за рецензенти
- Връзки към ресурси за документация и поддръжка
