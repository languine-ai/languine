# Конфигурация

Languine поддържа както TypeScript, така и JSON конфигурационни файлове за управление на вашата настройка за превод. Можете да изберете languine.config.ts за TypeScript или languine.json за JSON формат. И двата файла определят вашите изходни и целеви езици, както и файловете, които трябва да бъдат преведени.

---

## Основна конфигурация

Ето един основен пример в двата формата:

```typescript title="languine.config.ts"
import { defineConfig } from "languine";

export default defineConfig({
  locale: {
    source: "en",
    targets: ["es", "fr", "de"],
  },
  files: {
    json: {
      include: ["locales/[locale].json"],
    },
  },
});
```

```json title="languine.json"
{
  "locale": {
    "source": "en",
    "targets": ["es", "fr", "de"]
  },
  "files": {
    "json": {
      "include": ["locales/[locale].json"]
    }
  }
}
```

И двата формата поддържат същите опции за конфигурация. Изберете TypeScript за по-добра проверка на типовете и поддръжка на IDE, или JSON за простота и по-широка съвместимост с инструменти.

Нека разгледаме различните секции на тези файлове.

---

## Конфигурация на локализацията

Секцията за локализация определя вашите езици за превод:

- source: Вашият основен език за създаване на съдържание (напр. "en" за английски)
- targets: Масив от езици, на които искате да преведете (напр. ["es", "fr", "de"] за испански, френски и немски)

Languine поддържа широка гама от езикови кодове, включително:
- Прости кодове: en, es, fr, de
- Регионално специфични кодове: pt-BR, zh-CN, zh-TW

## Конфигурация на файловете

Секцията за файлове определя кои файлове Languine трябва да преведе. Можете да конфигурирате множество типове файлове и шаблони:

---

### JSON файлове
```typescript
files: {
  json: {
    include: ["locales/[locale].json"],
  },
}
```

---

### Markdown/MDX файлове
```typescript
files: {
  md: {
    include: ["docs/[locale]/*.md"],
  },
  mdx: {
    include: ["blog/[locale]/*.mdx"],
  },
}
```

---

### Файлове за мобилни платформи

За iOS (Xcode):
```typescript
files: {
  "xcode-xcstrings": {
    include: ["Example/Localizable.xcstrings"],
  },
  // Или за традиционни .strings файлове
  "xcode-strings": {
    include: ["Example/[locale].lproj/Localizable.strings"],
  },
}
```

---

За Android:
```typescript
files: {
  android: {
    include: ["locales/[locale].xml"],
  },
}
```

## Шаблони за файлове

Плейсхолдърът [locale] в шаблоните за файлове автоматично се заменя с подходящия езиков код по време на превода. Например:

- locales/[locale].json става:
  - locales/en.json
  - locales/es.json
  - locales/fr.json

Можете да включите множество шаблони за всеки тип файл и да използвате glob шаблони:
```typescript
files: {
  json: {
    include: [
      "src/locales/[locale].json",
      "src/components/[locale]/*.json"
    ],
  },
}
```

## Идентификатор на проекта (по желание)

Ако използвате облачните услуги на Languine, можете да включите идентификатора на вашия проект в конфигурацията:

```typescript
export default defineConfig({
  projectId: "prj_your_project_id",
  locale: {
    // ... конфигурация на локализацията
  },
  files: {
    // ... конфигурация на файловете
  },
});
```

## Проследяване на състоянието на превода

Languine автоматично създава и поддържа файл languine.lock, за да проследява състоянието на вашите преводи. Този файл:

- Проследява какво съдържание е било преведено
- Помага да се идентифицира какво трябва да бъде актуализирано, когато изходното съдържание се променя
- Трябва да бъде ангажиран с вашата система за контрол на версиите

Не е необходимо ръчно да редактирате файла за заключване - Languine го управлява автоматично, когато изпълнявате команди за превод.

## Най-добри практики

1. **Контрол на версиите**: Винаги ангажирайте както languine.config.ts, така и languine.lock в вашето хранилище
2. **Организация на файловете**: Дръжте файловете си за превод организирани в последователна структура на директориите
3. **Езикови кодове**: Използвайте стандартни езикови кодове, за да осигурите съвместимост
4. **Шаблони**: Използвайте специфични шаблони за включване, за да избегнете превеждането на ненужни файлове
5. **Идентификатор на проекта**: Дръжте идентификатора на проекта си в безопасност и не го споделяйте публично, ако използвате облачни услуги
