# Konfiguracija

Languine podržava i TypeScript i JSON konfiguracione datoteke za upravljanje vašim podešavanjima prevoda. Možete izabrati languine.config.ts za TypeScript ili languine.json za JSON format. Obe datoteke definišu vaše izvorne i ciljne jezike, kao i datoteke koje treba prevesti.

---

## Osnovna Konfiguracija

Evo osnovnog primera u oba formata:

```typescript title="languine.config.ts"
import { defineConfig } from "languine";

export default defineConfig({
  locale: {
    source: "en",
    targets: ["es", "fr", "de"],
  },
  files: {
    json: {
      include: ["locales/[locale].json"],
    },
  },
});
```

```json title="languine.json"
{
  "locale": {
    "source": "en",
    "targets": ["es", "fr", "de"]
  },
  "files": {
    "json": {
      "include": ["locales/[locale].json"]
    }
  }
}
```

Oba formata podržavaju iste opcije konfiguracije. Izaberite TypeScript za bolju proveru tipova i podršku IDE-u, ili JSON za jednostavnost i širu kompatibilnost alata.

Hajde da razložimo različite sekcije ovih datoteka.

---

## Konfiguracija Jezika

Sekcija jezika definiše vaše jezike za prevod:

- source: Vaš primarni jezik za kreiranje sadržaja (npr., "en" za engleski)
- targets: Niz jezika u koje želite da prevedete (npr., ["es", "fr", "de"] za španski, francuski i nemački)

Languine podržava širok spektar kodova jezika, uključujući:
- Jednostavni kodovi: en, es, fr, de
- Kodovi specifični za region: pt-BR, zh-CN, zh-TW

## Konfiguracija Datoteka

Sekcija datoteka definiše koje datoteke Languine treba da prevede. Možete konfigurisati više tipova i obrazaca datoteka:

---

### JSON Datoteke
```typescript
files: {
  json: {
    include: ["locales/[locale].json"],
  },
}
```

---

### Markdown/MDX Datoteke
```typescript
files: {
  md: {
    include: ["docs/[locale]/*.md"],
  },
  mdx: {
    include: ["blog/[locale]/*.mdx"],
  },
}
```

---

### Datoteke za Mobilne Platforme

Za iOS (Xcode):
```typescript
files: {
  "xcode-xcstrings": {
    include: ["Example/Localizable.xcstrings"],
  },
  // Ili za tradicionalne .strings datoteke
  "xcode-strings": {
    include: ["Example/[locale].lproj/Localizable.strings"],
  },
}
```

---

Za Android:
```typescript
files: {
  android: {
    include: ["locales/[locale].xml"],
  },
}
```

## Obrasci Datoteka

Placeholder [locale] u obrascima datoteka automatski se zamenjuje odgovarajućim kodom jezika tokom prevoda. Na primer:

- locales/[locale].json postaje:
  - locales/en.json
  - locales/es.json
  - locales/fr.json

Možete uključiti više obrazaca za svaki tip datoteke i koristiti glob obrasce:
```typescript
files: {
  json: {
    include: [
      "src/locales/[locale].json",
      "src/components/[locale]/*.json"
    ],
  },
}
```

## ID Projekta (Opcionalno)

Ako koristite Languine-ove cloud usluge, možete uključiti svoj ID projekta u konfiguraciju:

```typescript
export default defineConfig({
  projectId: "prj_your_project_id",
  locale: {
    // ... konfiguracija jezika
  },
  files: {
    // ... konfiguracija datoteka
  },
});
```

## Praćenje Stanja Prevoda

Languine automatski kreira i održava languine.lock datoteku za praćenje stanja vaših prevoda. Ova datoteka:

- Prati koji sadržaj je preveden
- Pomaže u identifikaciji šta treba ažurirati kada se izvorni sadržaj promeni
- Treba da bude uključena u vaš sistem kontrole verzija

Ne morate ručno uređivati lock datoteku - Languine je automatski upravlja kada pokrenete komande za prevod.

## Najbolje Prakse

1. **Kontrola Verzija**: Uvek uključite i languine.config.ts i languine.lock u vaš repozitorijum
2. **Organizacija Datoteka**: Držite svoje datoteke za prevod organizovane u doslednoj strukturi direktorijuma
3. **Kodovi Jezika**: Koristite standardne kodove jezika kako biste osigurali kompatibilnost
4. **Obrasci**: Koristite specifične obrasce uključivanja kako biste izbegli prevođenje nepotrebnih datoteka
5. **ID Projekta**: Držite svoj ID projekta sigurnim i ne delite ga javno ako koristite cloud usluge
