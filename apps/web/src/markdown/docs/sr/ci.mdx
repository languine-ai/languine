# CI/CD

Languine pruža CI/CD integraciju koja vam pomaže da prevedete svoju aplikaciju.

---

## Kako to funkcioniše

CI/CD radni tok u Languine-u prati strukturirani proces za automatizaciju ažuriranja prevoda u vašoj aplikaciji:

### 1. Inicijalizacija Akcije
- Radni tok počinje kada se akcija pokrene
- Postavlja potrebne Git konfiguracije i akreditive

### 2. Obrada Languine Motora
- Languine motor obrađuje sadržaj vaše aplikacije
- Obavlja zadatke prevođenja koristeći konfigurisana podešavanja

### 3. Detekcija Promena
Sistem proverava da li ima promena u prevodu i prati jedan od dva puta:

**Kada su Promene Pronađene:**
- Promene se komituju u repozitorijum
- Promene se šalju na određenu granu
- Kreira se Pull Request (PR)

**Kada Nema Pronađenih Promena:**
- Radni tok se završava bez kreiranja PR-a

### 4. Kreiranje Pull Request-a
Kada se detektuju promene, kreira se Pull Request sa:

**Sadržaj PR-a:**
- Jasan naslov koji ukazuje na ažuriranja prevoda
- Detaljan opis promena
- Potvrde o kvalitetu
- Uputstva za pregled
- Linkovi ka dokumentaciji i podršci

**Opis PR-a Uključuje:**
- Sažetak ažuriranja prevoda
- Spisak onoga što je promenjeno (ažurirani prevodi, doslednost terminologije)
- Sledeći koraci za recenzente
- Linkovi ka dokumentaciji i resursima podrške
