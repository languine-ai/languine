# Konfigurácia

Languine podporuje konfigurácie v TypeScript a JSON formáte na správu vašich prekladových nastavení. Môžete si vybrať buď languine.config.ts pre TypeScript alebo languine.json pre JSON formát. Obe súbory definujú vaše zdrojové a cieľové jazyky, ako aj súbory, ktoré je potrebné preložiť.

---

## Základná konfigurácia

Tu je základný príklad v oboch formátoch:

```typescript title="languine.config.ts"
import { defineConfig } from "languine";

export default defineConfig({
  locale: {
    source: "en",
    targets: ["es", "fr", "de"],
  },
  files: {
    json: {
      include: ["locales/[locale].json"],
    },
  },
});
```

```json title="languine.json"
{
  "locale": {
    "source": "en",
    "targets": ["es", "fr", "de"]
  },
  "files": {
    "json": {
      "include": ["locales/[locale].json"]
    }
  }
}
```

Obe formáty podporujú rovnaké možnosti konfigurácie. Vyberte si TypeScript pre lepšie typové overovanie a podporu IDE, alebo JSON pre jednoduchosť a širšiu kompatibilitu nástrojov.

Rozoberme rôzne sekcie týchto súborov.

---

## Konfigurácia jazyka

Sekcia jazyka definuje vaše prekladové jazyky:

- source: Váš primárny jazyk pre tvorbu obsahu (napr. "en" pre angličtinu)
- targets: Pole jazykov, do ktorých chcete prekladať (napr. ["es", "fr", "de"] pre španielčinu, francúzštinu a nemčinu)

Languine podporuje širokú škálu jazykových kódov, vrátane:
- Jednoduché kódy: en, es, fr, de
- Regionálne kódy: pt-BR, zh-CN, zh-TW

## Konfigurácia súborov

Sekcia súborov definuje, ktoré súbory by mal Languine preložiť. Môžete konfigurovať viacero typov súborov a vzorov:

---

### JSON súbory
```typescript
files: {
  json: {
    include: ["locales/[locale].json"],
  },
}
```

---

### Markdown/MDX súbory
```typescript
files: {
  md: {
    include: ["docs/[locale]/*.md"],
  },
  mdx: {
    include: ["blog/[locale]/*.mdx"],
  },
}
```

---

### Súbory pre mobilné platformy

Pre iOS (Xcode):
```typescript
files: {
  "xcode-xcstrings": {
    include: ["Example/Localizable.xcstrings"],
  },
  // Alebo pre tradičné .strings súbory
  "xcode-strings": {
    include: ["Example/[locale].lproj/Localizable.strings"],
  },
}
```

---

Pre Android:
```typescript
files: {
  android: {
    include: ["locales/[locale].xml"],
  },
}
```

## Vzory súborov

Zástupný symbol [locale] vo vzoroch súborov sa automaticky nahrádza príslušným jazykovým kódom počas prekladu. Napríklad:

- locales/[locale].json sa stáva:
  - locales/en.json
  - locales/es.json
  - locales/fr.json

Môžete zahrnúť viacero vzorov pre každý typ súboru a použiť glob vzory:
```typescript
files: {
  json: {
    include: [
      "src/locales/[locale].json",
      "src/components/[locale]/*.json"
    ],
  },
}
```

## ID projektu (voliteľné)

Ak používate cloudové služby Languine, môžete zahrnúť svoje ID projektu do konfigurácie:

```typescript
export default defineConfig({
  projectId: "prj_your_project_id",
  locale: {
    // ... konfigurácia jazyka
  },
  files: {
    // ... konfigurácia súborov
  },
});
```

## Sledovanie stavu prekladu

Languine automaticky vytvára a udržiava súbor languine.lock na sledovanie stavu vašich prekladov. Tento súbor:

- Sledovanie, ktorý obsah bol preložený
- Pomáha identifikovať, čo je potrebné aktualizovať, keď sa zmení zdrojový obsah
- Mal by byť zahrnutý do vášho systému správy verzií

Nemusíte manuálne upravovať súbor zamknutia - Languine ho spravuje automaticky, keď spustíte príkazy na preklad.

## Najlepšie praktiky

1. **Správa verzií**: Vždy commitujte oba súbory languine.config.ts a languine.lock do vášho repozitára
2. **Organizácia súborov**: Udržujte svoje prekladové súbory organizované v konzistentnej adresárovej štruktúre
3. **Jazykové kódy**: Používajte štandardné jazykové kódy na zabezpečenie kompatibility
4. **Vzory**: Používajte špecifické vzory zahrnutia, aby ste sa vyhli prekladu nepotrebných súborov
5. **ID projektu**: Udržujte svoje ID projektu v bezpečí a nezdieľajte ho verejne, ak používate cloudové služby
