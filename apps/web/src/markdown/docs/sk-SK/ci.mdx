# CI/CD

Languine poskytuje integráciu CI/CD, aby vám pomohol preložiť vašu aplikáciu.

---

## Ako to funguje

CI/CD pracovný postup v Languine sleduje štruktúrovaný proces na automatizáciu aktualizácií prekladu vo vašej aplikácii:

### 1. Inicializácia akcie
- Pracovný postup začína, keď je akcia spustená
- Nastaví potrebnú Git konfiguráciu a poverenia

### 2. Spracovanie Languine Engine
- Languine engine spracováva obsah vašej aplikácie
- Rieši úlohy prekladu pomocou nakonfigurovaných nastavení

### 3. Detekcia zmien
Systém kontroluje akékoľvek zmeny v preklade a nasleduje jednou z dvoch ciest:

**Keď sú nájdené zmeny:**
- Zmeny sú potvrdené do repozitára
- Zmeny sú odoslané na určenú vetvu
- Vytvorí sa Pull Request (PR)

**Keď nie sú nájdené zmeny:**
- Pracovný postup sa dokončí bez vytvorenia PR

### 4. Vytvorenie Pull Requestu
Keď sú zmeny detekované, vytvorí sa Pull Request s:

**Obsah PR:**
- Jasný názov indikujúci aktualizácie prekladu
- Podrobný popis zmien
- Potvrdenia o kontrole kvality
- Pokyny na revíziu
- Odkazy na dokumentáciu a podporu

**Popis PR obsahuje:**
- Zhrnutie aktualizácií prekladu
- Zoznam toho, čo sa zmenilo (aktualizované preklady, konzistencia terminológie)
- Ďalšie kroky pre recenzentov
- Odkazy na dokumentáciu a zdroje podpory
