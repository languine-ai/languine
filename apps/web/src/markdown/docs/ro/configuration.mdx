# Configurare

Languine suportă atât fișiere de configurare TypeScript, cât și JSON pentru a gestiona setările de traducere. Poți alege fie languine.config.ts pentru TypeScript, fie languine.json pentru format JSON. Ambele fișiere definesc limbile sursă și țintă, precum și fișierele care trebuie traduse.

---

## Configurare de bază

Iată un exemplu de bază în ambele formate:

```typescript title="languine.config.ts"
import { defineConfig } from "languine";

export default defineConfig({
  locale: {
    source: "en",
    targets: ["es", "fr", "de"],
  },
  files: {
    json: {
      include: ["locales/[locale].json"],
    },
  },
});
```

```json title="languine.json"
{
  "locale": {
    "source": "en",
    "targets": ["es", "fr", "de"]
  },
  "files": {
    "json": {
      "include": ["locales/[locale].json"]
    }
  }
}
```

Ambele formate suportă aceleași opțiuni de configurare. Alege TypeScript pentru o verificare mai bună a tipurilor și suport IDE, sau JSON pentru simplitate și compatibilitate mai largă cu instrumentele.

Să analizăm diferitele secțiuni ale acestor fișiere.

---

## Configurare Locală

Secțiunea locală definește limbile tale de traducere:

- source: Limba ta principală pentru crearea de conținut (de exemplu, "en" pentru engleză)
- targets: Un array de limbi în care dorești să traduci (de exemplu, ["es", "fr", "de"] pentru spaniolă, franceză și germană)

Languine suportă o gamă largă de coduri de limbă, inclusiv:
- Coduri simple: en, es, fr, de
- Coduri specifice regiunii: pt-BR, zh-CN, zh-TW

## Configurare Fișiere

Secțiunea fișiere definește care fișiere ar trebui traduse de Languine. Poți configura mai multe tipuri de fișiere și modele:

---

### Fișiere JSON
```typescript
files: {
  json: {
    include: ["locales/[locale].json"],
  },
}
```

---

### Fișiere Markdown/MDX
```typescript
files: {
  md: {
    include: ["docs/[locale]/*.md"],
  },
  mdx: {
    include: ["blog/[locale]/*.mdx"],
  },
}
```

---

### Fișiere pentru Platforme Mobile

Pentru iOS (Xcode):
```typescript
files: {
  "xcode-xcstrings": {
    include: ["Example/Localizable.xcstrings"],
  },
  // Sau pentru fișiere .strings tradiționale
  "xcode-strings": {
    include: ["Example/[locale].lproj/Localizable.strings"],
  },
}
```

---

Pentru Android:
```typescript
files: {
  android: {
    include: ["locales/[locale].xml"],
  },
}
```

## Modele de Fișiere

Placeholderul [locale] din modelele de fișiere este înlocuit automat cu codul de limbă corespunzător în timpul traducerii. De exemplu:

- locales/[locale].json devine:
  - locales/en.json
  - locales/es.json
  - locales/fr.json

Poți include mai multe modele pentru fiecare tip de fișier și folosi modele glob:
```typescript
files: {
  json: {
    include: [
      "src/locales/[locale].json",
      "src/components/[locale]/*.json"
    ],
  },
}
```

## ID Proiect (Opțional)

Dacă folosești serviciile cloud ale Languine, poți include ID-ul proiectului tău în configurație:

```typescript
export default defineConfig({
  projectId: "prj_your_project_id",
  locale: {
    // ... configurare locală
  },
  files: {
    // ... configurare fișiere
  },
});
```

## Urmărirea Stării Traducerii

Languine creează și menține automat un fișier languine.lock pentru a urmări starea traducerilor tale. Acest fișier:

- Urmărește ce conținut a fost tradus
- Ajută la identificarea a ceea ce trebuie actualizat atunci când conținutul sursă se schimbă
- Ar trebui să fie inclus în sistemul tău de control al versiunilor

Nu trebuie să editezi manual fișierul de blocare - Languine îl gestionează automat atunci când rulezi comenzi de traducere.

## Cele Mai Bune Practici

1. **Controlul Versiunii**: Întotdeauna comite atât languine.config.ts, cât și languine.lock în repository-ul tău
2. **Organizarea Fișierelor**: Păstrează fișierele tale de traducere organizate într-o structură de directoare consistentă
3. **Coduri de Limbă**: Folosește coduri de limbă standard pentru a asigura compatibilitatea
4. **Modele**: Folosește modele de includere specifice pentru a evita traducerea fișierelor inutile
5. **ID Proiect**: Păstrează ID-ul proiectului tău în siguranță și nu-l împărtăși public dacă folosești servicii cloud
