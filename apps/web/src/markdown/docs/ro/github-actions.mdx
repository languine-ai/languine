# GitHub Actions

Cu acțiunea GitHub, poți traduce cu ușurință codul tău doar prin împingerea codului în depozit, acțiunea GitHub va traduce apoi noile tale chei, fie va crea o cerere de extragere, fie va comite modificările tale.
Acest lucru îți economisește mult timp și efort, menținând în același timp traduceri de înaltă calitate.

---

## Instalare

Pentru a folosi Acțiunea Languine GitHub în depozitul tău, va trebui să:

1. Configurezi secretele depozitului tău cu acreditivele Languine
2. Creezi un fișier de flux de lucru în depozitul tău

### Configurarea Secretelor Depozitului

Adaugă următoarele secrete în setările depozitului tău (Setări > Secrete și variabile > Acțiuni):

- `LANGUINE_API_KEY`: Cheia ta API Languine
- `LANGUINE_PROJECT_ID`: ID-ul proiectului tău Languine

Poți găsi aceste valori în tabloul de bord Languine sub Setările Proiectului.



### Variabile de Mediu

Acțiunea GitHub folosește următoarele variabile de mediu:
- `LANGUINE_API_KEY` (Obligatorie): Cheia ta API Languine pentru autentificare
- `LANGUINE_PROJECT_ID` (Obligatorie): ID-ul proiectului tău Languine 
- `GITHUB_TOKEN` (Obligatorie): Furnizată automat de GitHub, utilizată pentru crearea cererilor de extragere

### Exemplu de Flux de Lucru

Creează un nou fișier .github/workflows/languine.yml în depozitul tău cu următorul conținut:

```ts title=".github/workflows/release.yml"
name: Deploy Website

on:
  push:
    branches:
      - '**'
    paths:
      - apps/web/**

jobs:
  translate:
    runs-on: ubuntu-latest
    permissions:
      contents: write
      pull-requests: write
    steps:
      - uses: actions/checkout@v4
      
      - name: Languine Action
        uses: languine/languine-action@v1
        env:
           GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
        with:
          api-key: ${{ secrets.LANGUINE_API_KEY }}
          project-id: ${{ secrets.LANGUINE_PROJECT_ID }}
```
 
 Acest flux de lucru va:
- Rula la împingeri pe ramura principală
- Rula la cereri de extragere pe ramura principală
- Poate fi declanșat manual din tab-ul Acțiuni
- Verifica codul tău și va rula procesul de traducere Languine
- Crea o cerere de extragere cu orice noi traduceri (configurabil)

#### Opțiunea Cerere de Extragere

Opțiunea `create-pull-request` controlează dacă acțiunea creează o cerere de extragere cu modificările sau le comite direct în ramură:

```typescript title=".github/workflows/languine.yml"
with:
  create-pull-request: true # creează o cerere de extragere cu modificările
```

Când este setată pe `true`, acțiunea va:
- Crea o nouă ramură cu modificările
- Crea o cerere de extragere pentru revizuire
- Adăuga etichete și descriere adecvate

Când este setată pe `false` (implicit), acțiunea va:
- Comite modificările direct în ramura curentă
- Împinge modificările fără a crea o cerere de extragere
